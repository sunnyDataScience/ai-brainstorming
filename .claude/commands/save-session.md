# 儲存 Session 指令

擷取本次 session 中發生的所有事 -- 建構了什麼、什麼有效、什麼失敗、還剩什麼 -- 寫入有日期的檔案，讓下次 session 可以從中斷處繼續。

## 使用時機

- 工作 session 結束前，關閉 Claude Code 之前
- 接近 context 限制前（先執行此指令，再開新 session）
- 解決複雜問題後想記住的時候
- 需要將上下文傳遞給未來 session 的任何時候

## 流程

### 步驟 1: 收集上下文

寫入檔案前收集：
- 讀取本次 session 修改的所有檔案（用 git diff 或回憶對話）
- 審查討論、嘗試和決定的內容
- 記錄遇到的錯誤及解決方式
- 檢查當前測試/建置狀態

### 步驟 2: 建立 sessions 資料夾

```bash
mkdir -p ~/.claude/sessions
```

### 步驟 3: 寫入 session 檔案

建立 `~/.claude/sessions/YYYY-MM-DD-<short-id>-session.tmp`

### 步驟 4: 填入所有區段

誠實地寫每個區段。不跳過區段 -- 如果某區段真的沒內容就寫「無」。

### 步驟 5: 向使用者展示

```
Session 已儲存至 [實際檔案路徑]

這看起來正確嗎？關閉前有什麼要修正或補充的嗎？
```

## Session 檔案格式

```markdown
# Session: YYYY-MM-DD

**開始時間:** [大約時間]
**最後更新:** [當前時間]
**專案:** [專案名稱或路徑]
**主題:** [一行摘要]

---

## 我們在建構什麼
[1-3 段描述功能、bug 修復或任務]

---

## 確認有效的部分（含證據）
- **[有效的事項]** -- 確認方式: [具體證據]

---

## 未成功的部分（及原因）
- **[嘗試的方法]** -- 失敗原因: [確切原因/錯誤訊息]

---

## 尚未嘗試的方法
- [方法/想法]

---

## 檔案當前狀態

| 檔案 | 狀態 | 備註 |
|------|------|------|
| `path/to/file.ts` | 完成 | [說明] |
| `path/to/file.ts` | 進行中 | [已完成/待完成] |
| `path/to/file.ts` | 壞掉 | [問題] |

---

## 已做的決策
- **[決策]** -- 原因: [選擇此方案而非替代方案的理由]

---

## 阻礙與待解決問題
- [阻礙/待解決問題]

---

## 確切的下一步
[恢復時要做的第一件事]
```

## 注意事項

- 每個 session 有自己的檔案 -- 絕不附加到之前 session 的檔案
- 「未成功的部分」最關鍵 -- 沒有它未來 session 會盲目重試失敗方法
- 此檔案設計為在下次 session 開始時由 Claude 讀取
