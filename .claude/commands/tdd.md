# TDD 指令

此指令呼叫 **tdd-guide** agent 來強制執行測試驅動開發方法論。

## 功能說明

1. **建立介面** - 先定義 type/interface
2. **先產生測試** - 撰寫失敗測試 (RED)
3. **實作最小程式碼** - 只寫足以通過的程式碼 (GREEN)
4. **重構** - 在測試保持綠燈的情況下改善程式碼 (REFACTOR)
5. **驗證覆蓋率** - 確保 80%+ 測試覆蓋率

## 使用時機

- 實作新功能
- 新增函式/元件
- 修復 bug（先寫重現 bug 的測試）
- 重構現有程式碼
- 建構關鍵商業邏輯

## TDD 循環

```
RED -> GREEN -> REFACTOR -> REPEAT

RED:      寫一個失敗測試
GREEN:    寫最小程式碼讓測試通過
REFACTOR: 改善程式碼，保持測試通過
REPEAT:   下一個功能/場景
```

## 最佳實踐

**應該做:**
- 在任何實作之前先寫測試
- 執行測試確認失敗後再實作
- 寫最小程式碼讓測試通過
- 只在測試綠燈後才重構
- 加入邊界情況和錯誤場景
- 關鍵程式碼目標 100% 覆蓋率

**不應該做:**
- 先寫實作再寫測試
- 跳過每次變更後的測試執行
- 一次寫太多程式碼
- 忽略失敗的測試
- 測試實作細節（應測試行為）
- mock 所有東西（優先整合測試）

## 覆蓋率要求

- **80% 最低**適用於所有程式碼
- **100% 要求**適用於：財務計算、認證邏輯、安全關鍵程式碼、核心商業邏輯

## 與其他指令的搭配

- 先用 `/plan` 了解要建構什麼
- 用 `/tdd` 以測試驅動方式實作
- 如有建置錯誤用 `/build-fix`
- 用 `/review-code` 審查實作
- 用 `/verify` 驗證覆蓋率
