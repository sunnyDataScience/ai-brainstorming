## 審查前檢查

- [ ] 程式碼可編譯無錯誤
- [ ] 所有測試通過
- [ ] 符合專案風格規範
- [ ] 文檔已更新
- [ ] 已完成自我審查

## 審查重點

### 1. 程式碼品質
- **可讀性**: 程式碼是否容易理解?
- **可維護性**: 是否容易修改?
- **一致性**: 是否遵循專案慣例?
- **複雜度**: 複雜部分是否有文檔?

### 2. 架構與設計
- **SOLID 原則**: 是否遵循?
- **設計模式**: 是否使用適當模式?
- **關注點分離**: 職責是否明確劃分?
- **API 設計**: 介面是否乾淨直覺?

### 3. 效能與安全
- **效能**: 是否有明顯瓶頸?
- **安全**: 是否遵循安全最佳實踐?
- **資源使用**: 記憶體/CPU 使用是否合理?
- **錯誤處理**: 是否覆蓋邊界情況?

## 重構時機

- 偵測到 code smells
- 效能問題浮現
- 新增功能變得困難
- 技術債累積過多

## 重構策略

| 策略 | 適用場景 |
| :--- | :--- |
| Extract Method | 函式過長，有可複用邏輯 |
| Extract Variable | 條件表達式過複雜 |
| Replace Conditional with Polymorphism | 多重 if/switch |
| Introduce Parameter Object | 參數過多 |
| Move Method | 方法在錯誤的類別中 |
