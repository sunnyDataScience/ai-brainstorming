## 1. 文檔類型

| 類型 | 內容 | 格式 |
| :--- | :--- | :--- |
| **API 文檔** | OpenAPI 規範、端點說明、認證、錯誤碼 | OpenAPI YAML |
| **架構文檔** | 系統概覽、元件圖、資料流、基礎設施 | Markdown + Mermaid |
| **使用者文檔** | 快速開始、教學、操作指南 | Markdown |
| **開發者文檔** | 環境設置、風格指南、貢獻規範 | Markdown |

## 2. 文檔即程式碼

### 目錄結構

```
docs/
├── api/              # API 規範
│   ├── openapi.yaml
│   └── examples/
├── architecture/     # 架構文檔
│   ├── overview.md
│   └── diagrams/
├── guides/           # 操作指南
│   ├── getting-started.md
│   └── tutorials/
└── developer/        # 開發者文檔
    ├── setup.md
    └── contributing.md
```

### 撰寫規範

- **簡潔明瞭**: 直接切入重點
- **主動語態**: 「設定伺服器」而非「伺服器應被設定」
- **包含範例**: 可運行的真實範例
- **定期更新**: 保持資訊最新
- **版本控制**: 追蹤變更

## 6. 最佳實踐

1. **隨開發同步撰寫**: 不要事後補文檔
2. **文檔也要 Review**: 納入 Code Review 流程
3. **人人有責**: 不是特定人的工作
4. **持續改善**: 定期收集回饋並優化
