## A. 核心安全原則

- [ ] **最小權限**: 元件和使用者僅授予必要權限
- [ ] **縱深防禦**: 多層安全控制，無單點失效
- [ ] **預設安全**: 預設配置是安全的
- [ ] **攻擊面最小化**: 關閉不必要的端口、功能、API

## B. 資料安全與隱私

### 資料分類與收集
- [ ] 所有資料依敏感性分類 (公開/內部/機密/PII)
- [ ] 只收集業務必要資料
- [ ] PII 收集前已獲使用者同意

### 傳輸安全
- [ ] 外部通訊使用 TLS 1.2+

### 儲存安全
- [ ] 敏感資料加密儲存 (AES-256)
- [ ] 金鑰安全管理 (KMS/Vault)
- [ ] 備份資料同等加密保護

### 資料生命週期
- [ ] 日誌避免記錄敏感資訊 (已遮罩/脫敏)
- [ ] 定義資料保留期限

## C. 應用程式安全

### 認證
- [ ] 密碼使用強 hash (Argon2/bcrypt)
- [ ] Session Token 安全 (HttpOnly, Secure, 超時失效)
- [ ] 有速率限制/帳戶鎖定防暴力破解

### 授權
- [ ] 物件級別授權 (使用者 A 無法存取使用者 B 資料)
- [ ] 功能級別授權 (敏感操作有權限檢查)

### 輸入驗證與輸出編碼
- [ ] 防注入: 參數化查詢/ORM
- [ ] 防 XSS: 上下文編碼 + CSP
- [ ] 防 CSRF: SameSite Cookie / CSRF Token

### API 安全
- [ ] 所有端點經認證授權
- [ ] 速率限制
- [ ] 嚴格參數白名單驗證
- [ ] 回應僅包含必要資料

### 依賴安全
- [ ] 定期漏洞掃描 (npm audit/Snyk/Dependabot)
- [ ] 高危漏洞及時更新
