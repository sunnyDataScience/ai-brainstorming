# 積木模組登記表 (Module Registry)

> 網站不是頁面集合，而是一組可以重組的功能積木。
> `Website = Σ (Module_i × Depth_i)`

---

## 深度分級說明


| 等級     | 符號  | 意涵                     | 適用場景               |
| ------ | --- | ---------------------- | ------------------ |
| L1 基礎  | 🟡  | 最小可用版，通常用第三方服務或現成元件    | MVP / Landing Page |
| L2 標準  | 🟠  | 客製化程度中等，有自己的 UI + 基本邏輯 | 成長期產品              |
| L3 企業級 | 🔴  | 高度客製化，含進階邏輯、多角色、可擴充架構  | 規模化 / B2B          |


---

## 模組 1：Identity（身份系統）

> 所有長期關係型網站的起點 — 知道「誰」在用

### 功能清單


| 深度  | 功能                 | 說明                         |
| --- | ------------------ | -------------------------- |
| L1  | Email 註冊/登入        | 基本帳號系統                     |
| L1  | OAuth 社群登入         | Google / GitHub / Facebook |
| L1  | Session 管理         | 登入狀態維持                     |
| L2  | 角色權限 (RBAC)        | 管理員 / 編輯者 / 一般用戶           |
| L2  | 個人檔案               | 頭像、暱稱、偏好設定                 |
| L2  | 訂閱狀態綁定             | 付費/免費用戶區分                  |
| L3  | 多租戶 (Multi-tenant) | B2B 組織管理                   |
| L3  | SSO / SAML         | 企業單一登入                     |
| L3  | MFA 雙因素驗證          | 安全性要求高的場景                  |
| L3  | API Key 管理         | 開發者平台                      |


### 對應頁面建議

- 登入/註冊頁、個人設定頁、（L2+）角色管理頁、（L3）組織管理頁

### 代表服務

- Supabase Auth / Firebase Auth / Auth0 / Clerk
