# 品質檢查清單 (Quality Checklist)

> 通用驗收清單，適用於所有網站類型。對照你的 Design System 填入具體數值。
> 對應 `guides/vibe_coding_build_strategy.md` → Step 7。

---

## 一、設計一致性檢查

### 1.1 視覺一致性

- [ ] **色彩系統**
  - [ ] 所有顏色來自 Design Tokens（無隨意色值）
  - [ ] Primary / Secondary / Accent 使用正確
  - [ ] 語意色（Success / Warning / Error / Info）用途正確
  - [ ] 中性色階層分明（背景/邊框/文字）
  - [ ] 無未定義的顏色使用

- [ ] **字體系統**
  - [ ] 字體 family 與 Design System 一致
  - [ ] 字級遵循定義的階梯（Display → H1 → H2 → H3 → Body → Small → Caption）
  - [ ] 字重使用正確（標題 600-700 / 正文 400）
  - [ ] 行高設定正確

- [ ] **元件風格**
  - [ ] 圓角值與 Design Tokens 一致
  - [ ] 陰影使用正確層級（SM / MD / LG）
  - [ ] 邊框樣式統一
  - [ ] 間距使用 Spacing Scale（不隨意設值）

### 1.2 佈局一致性

- [ ] **間距系統**
  - [ ] 使用定義的 Spacing Scale
  - [ ] 元件間距/區塊間距/頁面間距層級分明
  - [ ] 無隨意間距值

- [ ] **響應式設計**
  - [ ] Desktop 佈局正確
  - [ ] Tablet 適配良好
  - [ ] Mobile 適配良好（如支援）
  - [ ] 各斷點之間的轉換平滑
  - [ ] 內容不會溢出或斷行

---

## 二、功能完整性檢查

### 2.1 頁面功能

- [ ] **基本功能**
  - [ ] Primary goal 達成
  - [ ] Secondary goal 達成（如有）
  - [ ] 所有必要區塊都存在
  - [ ] 導航路徑清晰

- [ ] **互動功能**
  - [ ] Hover 狀態正確顯示
  - [ ] Focus 狀態可見（含 focus ring）
  - [ ] Active 狀態回饋
  - [ ] Disabled 狀態合理（灰度 + 原因提示）
  - [ ] 點擊/輸入響應正常

### 2.2 狀態處理

- [ ] **Loading 狀態**
  - [ ] Skeleton screen 或 spinner 顯示
  - [ ] 長任務有進度指示
  - [ ] 無內容閃爍（保留上次成功狀態）

- [ ] **Error 狀態**
  - [ ] 錯誤訊息友善、具體
  - [ ] 提供重試或替代方案
  - [ ] 錯誤不會導致頁面崩潰
  - [ ] 表單驗證訊息格式統一

- [ ] **Empty 狀態**
  - [ ] 無資料時顯示引導
  - [ ] 提供建議操作（CTA）
  - [ ] 視覺設計友善

## 三、內容品質檢查

### 3.1 文案品質

- [ ] **語氣一致**
  - [ ] 符合 Brand & Tone Guidelines
  - [ ] 無偏離品牌語氣的文案
  - [ ] 術語使用準確且統一

- [ ] **語言規範**
  - [ ] 語言設定正確（中/英/雙語）
  - [ ] 技術術語處理一致
  - [ ] 無錯字或語法錯誤

### 3.2 資料呈現

- [ ] **數據格式**
  - [ ] 日期格式與 Data Pattern 一致
  - [ ] 數字格式正確（千分位/小數/百分比）
  - [ ] 單位標示清楚
  - [ ] 貨幣格式正確

---

## 四、技術實作檢查

### 4.1 程式碼品質

- [ ] **技術棧**
  - [ ] 使用專案指定的技術棧
  - [ ] 無使用禁用的函式庫
  - [ ] 依賴管理正確

- [ ] **程式碼規範**
  - [ ] 元件命名有意義且一致
  - [ ] 檔案結構清晰
  - [ ] 無重複代碼（DRY）

### 4.2 效能指標

- [ ] **載入效能**
  - [ ] 首次載入符合目標
  - [ ] 互動響應 < 100ms
  - [ ] 動畫流暢
  - [ ] 無記憶體洩漏

- [ ] **優化實作**
  - [ ] 圖片適當壓縮/延遲載入
  - [ ] 大資料使用虛擬滾動（如適用）
  - [ ] 元件適當拆分
  - [ ] 避免過度渲染

## 五、無障礙與 SEO

### 5.1 無障礙支援

- [ ] **鍵盤操作**
  - [ ] Tab 順序合理
  - [ ] Enter/Space 可觸發互動元素
  - [ ] Escape 可關閉 Modal/Dropdown
  - [ ] 無鍵盤陷阱

- [ ] **螢幕閱讀器**
  - [ ] 適當的 ARIA 標籤
  - [ ] 圖片有 alt 文字
  - [ ] 表單標籤關聯正確
  - [ ] 色彩對比 ≥ 4.5:1（WCAG AA）

### 5.2 SEO（如為公開頁面）

- [ ] Meta title / description 設定
- [ ] Open Graph / Twitter Card 設定
- [ ] 語意化 HTML（h1-h6 結構正確）
- [ ] Schema markup（如適用）

## 七、最終確認

### 7.1 整體評估

- [ ] **Design System 遵循度**：≥ 95%
- [ ] **功能完成度**：100%（所有 Page Spec 需求）
- [ ] **狀態覆蓋度**：100%（Loading / Error / Empty / Disabled）
- [ ] **RWD 覆蓋度**：所有定義的斷點
- [ ] **效能達標**：符合 TECH & CONSTRAINT LAYER 定義

### 7.2 問題分級

| 等級 | 定義 | 處理 |
|------|------|------|
| Critical | 阻礙核心功能或嚴重違反 Design System | 必須立即修復 |
| Major | 影響體驗但有 workaround | 應排程修復 |
| Minor | 細節優化，不影響使用 | 可下次迭代修 |

### 7.3 通過標準

- **通過**：Critical = 0，Major ≤ 3
- **有條件通過**：Critical = 0，Major ≤ 5，需排程修正
- **不通過**：有 Critical 或 Major > 5
