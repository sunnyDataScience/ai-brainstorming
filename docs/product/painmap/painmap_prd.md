## 前言：本文件的由來

本 PRD 從 FirstDollar PRD 的上游缺口出發。

FirstDollar 解決「有模糊點子 → 72 小時拿到第一塊錢」的執行問題。但在 FirstDollar 之前，存在一個更根本的問題：**有能力的人找不到值得做的題目，或看不清問題的本質。**

**輸入材料**：
1. `first_principles_playbook.md` — 第一性原理方法論，定義四個物理量
2. `firstdollar_prd.md` — FirstDollar PRD，5 位 persona 訪談揭示的上游痛點
3. `firstdollar_strategy_addendum.md` — 護城河與飛輪分析，Offer Vault 模式的泛化可能
4. Christensen 破壞式創新理論 — 作為 GTM 策略工具化的理論基礎

**核心發現**：
- 阿傑「電腦裡有 40 個 idea.md」— 不缺點子，缺**結構化篩選**
- David「隱性知識無法資產化」— 有行業洞察，缺**轉化成產品的框架**
- Kai「粉絲需求爛在留言區」— 痛點就在眼前，缺**結構化提取工具**
- 現行工具（IdeaCheck）跳過結構化直接打分 — 方法論層級的空白

## 0. 市場調查摘要（承接 FirstDollar Persona 洞察）

### 0.1 從 FirstDollar 五位 persona 提取的上游痛點

| Persona | FirstDollar 痛點（執行層） | **PainMap 痛點（找題層）** | 願付金額 |
| :--- | :--- | :--- | :--- |
| **阿傑**（32，資深後端 indie hacker） | 做到 60% 就棄坑 | 40 個 idea.md 不知道做哪個；看不出哪個痛點「值得做 3 個月」 | $49–99 USD/月（for 幫我砍到剩 1 個） |
| **淑芬**（45，電商手作媽媽） | 不會行銷、不會廣告 | 不知道新品要試什麼方向；客人的需求散在 LINE 群裡沒人整理 | NT$300–500/月 |
| **Vivian**（29，Growth PM wantrepreneur） | 卡在 30% 就棄坑 | 做了 3 年 PM 但不確定哪個行業痛點是「我能切的」；怕選錯題目浪費半年 | $49–79/月 |
| **Kai**（27，AI 工具 YouTuber） | 有受眾沒有手 | 8.5 萬訂閱者的留言是一座金礦，但他不知道怎麼從中提取可商品化的痛點 | 為「省我 10 小時整理留言」付 $99 |
| **David**（38，UX 顧問） | 接案機會成本殺死 side project | 10 年顧問經驗看過 50 個行業痛點，但不知道哪個值得從顧問模式轉成產品模式 | 為「行業痛點資料庫 + GTM 路徑」今天刷卡 |

### 0.2 跨越性共識（PainMap 專屬強訊號）

1. **「不缺點子，缺結構」** — 5/5 persona 都有痛點素材（留言、客服、行業經驗），但沒有工具把散亂素材結構化成可評估的問題格式
2. **「怕選錯」比「不知道怎麼做」更痛** — Vivian 和阿傑的棄坑根因不是執行力不足，是**對題目沒信心**。信心不來自分數，來自結構化的證據鏈
3. **「行業知識是隱性的」** — David 腦中有 50 個行業痛點，但它們以「經驗」形式存在，無法被搜尋、比較、傳遞。一旦結構化，這些知識本身就是產品
4. **「別人驗證過的痛點」比「AI 分析報告」有說服力** — 所有 persona 都不信 AI 打分，但都表示願意看「跟我類似的人實際驗證過什麼」

### 0.3 UX 角度的市場洞察

| # | 洞察 | 設計含義 |
| :--- | :--- | :--- |
| UX-1 | **散亂痛點 → 結構化是高認知負荷任務，現有工具零支援** | 產品核心 UX 必須把「輸入散亂素材 → 輸出結構化問題」的認知成本壓到最低。不是問卷，是對話式引導 |
| UX-2 | **「選題恐懼」的根因是缺乏比較基準** | 產品必須提供「其他人在同行業驗證過什麼」的參照系。不是排名，是地圖 |
| UX-3 | **行業 know-how 是最被低估的創業資產** | 產品要幫使用者把「我知道但說不出來」的隱性知識外化成可操作的問題定義 |
| UX-4 | **現有工具的「從零開始」假設是錯的** | 多數使用者不是從零開始想點子，而是已經有散亂的素材（留言、對話、經驗）。產品入口應該是「匯入你已有的東西」而非「輸入一句話 idea」 |

### 0.4 產品命題一句話

> **PainMap 是一個行業痛點結構化引擎。我們不幫你想點子、不打分數；而是把你已經看見但還沒整理的散亂痛點，結構化成可驗證的問題定義 + 破壞式 GTM 切入路徑，並讓這些結構化痛點匯入一張越用越準的行業痛點圖譜。**

## 4. 範圍與限制

| 項目 | 內容 |
| :--- | :--- |
| **MVP 功能範圍（≤ 4 個核心）** | 1. **痛點採集器**（raw text / CSV / 圖片 OCR → 結構化痛點）<br>2. **問題本質拆解**（物理量檢驗 + affinity clustering + 一句話問題本質）<br>3. **破壞式 GTM 策略**（Strategy Canvas + 低端切入方案 + GTM 一頁紙）<br>4. **痛點圖譜 v0**（先收資料、可瀏覽，推薦功能 v1.1） |
| **非功能需求** | **性能**：痛點結構化 ≤ 90 秒（10 條以內）<br>**安全**：使用者原始資料預設不用於模型訓練；匿名化流程不可逆<br>**可用性**：99.5% SLA（MVP）；i18n zh-Hant 預設，en 次之<br>**隱私**：圖譜貢獻 opt-in only；可隨時撤回 |
| **不做什麼（救命清單）** | 1. **不做 idea scoring** — 品牌基因，不可妥協<br>2. **不做「100 個 AI 點子產生器」** — 使用者不缺點子，缺結構<br>3. **不做市場報告 / TAM 預測** — AI 5 分鐘可生，無增量價值<br>4. **不做 App** — Web-first<br>5. **不做 B2B** — 焦點在 solo / small team<br>6. **不做 AI 替你決定** — 所有「做 or 不做」的判斷留給人<br>7. **不做商業計畫書生成** — David 說那是「結構正確但缺 context 的廢話」<br>8. **不做 pitch deck 工具** — 在第一塊錢之前，pitch deck 是代理動作 |
| **假設與依賴** | **假設 1**：使用者願意花 15–30 分鐘完成結構化流程（高風險，需 Private Beta 驗證）<br>**假設 2**：匿名化痛點圖譜對新使用者有吸引力（需冷啟動種子資料驗證）<br>**假設 3**：與 FirstDollar 串接能提升「結構化問題→第一塊錢」的轉化率 ≥ 2x<br>**依賴**：LLM（Claude Sonnet 4.6）、Supabase（DB + pgvector）、Vercel（hosting）、FirstDollar API（串接） |
| **定價草案** | **Free**：結構化 3 個問題 / 月 + 瀏覽圖譜前 10 條<br>**Pro** NT$590/月：unlimited 結構化 + 完整圖譜 + GTM 策略 + FirstDollar 串接<br>**Expert** NT$1,290/月：+ 蘇格拉底對話引導（深度知識外化）+ 私人痛點知識庫 + 優先圖譜推薦<br>**承諾**：月付不綁約、隨時停用 |

## 附錄 A：與 FirstDollar 的功能對照

| 維度 | PainMap | FirstDollar | 關係 |
| :--- | :--- | :--- | :--- |
| 解決的問題 | 不知道做什麼 / 看不清問題本質 | 知道做什麼但做不出手 | 上下游 |
| 核心交付 | 結構化問題 + GTM 路徑 | 7 天 Sprint + 第一塊錢 | PainMap 輸出 = FirstDollar 輸入 |
| North Star | 結構化問題 → 付費訊號轉化率 | $ / 週 | 串接指標 |
| 護城河 | 行業痛點圖譜 | Offer Template Vault | 互補 |
| 飛輪 | 痛點越多 → 圖譜越準 → 找題越容易 | Offer 越多 → 建議越準 → 成交越高 | 雙飛輪 |
| 反焦慮 | 禁止 idea score、用「問題」不用「點子」 | 禁止 idea score、只追 $ | 同一基因 |

## 附錄 B：與 IdeaCheck / ShipYourIdea 的競品差異

| 維度 | IdeaCheck / ShipYourIdea | PainMap |
| :--- | :--- | :--- |
| 輸入 | 一句話 idea 描述 | 散亂素材（留言、對話、經驗、CSV） |
| 過程 | AI 打分 + 報告 | 結構化拆解 + 物理量檢驗 + 策略設計 |
| 輸出 | 分數 + 分析報告 | 結構化問題 + GTM 一頁紙 + 下一步行動 |
| 資料資產 | 無（每次獨立運算） | 行業痛點圖譜（群體智慧累積） |
| 使用者心智 | 「我的點子幾分？」 | 「這個問題的本質是什麼？」 |
| 行動驅動力 | 看完報告 → 更焦慮 | 看到缺口 → 知道下一步 |
