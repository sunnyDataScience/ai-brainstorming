# PainMap 破壞式創新引導流程 — 課堂工作坊版

> **適用對象**：講師帶領初學者，從 0→1 用 AI 工具完成痛點結構化到 GTM 行動計畫
> **適用平台**：任何 AI 聊天工具（ChatGPT、Claude、Gemini、Copilot 等）
> **所需時間**：完整 7 步約 3–5 小時（可分多堂完成）
> **示範案例**：全程使用「Shopee 小賣家客服」場景貫穿

---

## 資料夾結構

```
skills/
├── README.md              ← 你在這裡
├── 00_guide.md            ← 原始使用指南（完整版）
├── 01–07_*.md             ← 原始 skill 檔案（完整版，含所有 prompt + 說明）
│
├── teacher/               ← 老師示範用
│   └── stepXX_*/
│       ├── prompt.md          填好範例的 prompt（直接貼入 AI）
│       └── output_sample.md   擬真 AI 回覆 + 💡老師提醒標注
│
└── student/               ← 學生演練用
    └── stepXX_*/
        ├── prompt.md          帶 {{空白}} 的 prompt + 操作指引 + 檢查點
        └── output_reference.md 產出範本 + 品質對照表
```

## 課堂流程

### 每一步的上課節奏

```
① 老師示範（10-15 分鐘）
   打開 teacher/stepXX/prompt.md → 貼入 AI → 展示結果
   對照 output_sample.md 講解重點（💡 標注處）

② 學生演練（15-30 分鐘）
   打開 student/stepXX/prompt.md → 填入自己的素材 → 貼入 AI
   對照 output_reference.md 自我檢查產出品質

③ 小組討論（5-10 分鐘）
   比對彼此的產出，看誰的「物理量」最具體
   老師點評 1-2 組常見問題
```

### 7 步流程總覽

| Step | Phase | 主題 | 老師示範重點 | 預估時間 |
|------|-------|------|-------------|---------|
| 01 | 痛點採集 | 散亂素材 → 結構化痛點 | AI 猜的 vs 素材確認的差異 | 30-60 min |
| 02 | 問題本質拆解 | 痛點聚合 → 問題主題 | 物理量畫布的手動填寫 | 30-45 min |
| 03 | 破壞式機會辨識 | 低端 vs 新市場破壞 | 何時質疑 AI 的樂觀分析 | 20-30 min |
| 04 | 策略圖設計 | Strategy Canvas + 紅隊 | 曲線形狀不同 vs 全部更低 | 30-45 min |
| 05 | 最小產品設計 | 手工交付 + 定價 + 預售 | 為什麼不寫程式才是重點 | 30-45 min |
| 06 | 不對稱動機檢驗 | 大公司會不會來搶 | 安全 ✅ 的真正意思 | 15-20 min |
| 07 | GTM 一頁紙 | 72 小時行動計畫 | Day 1 必須 5 分鐘內能開始 | 20-30 min |

## 示範案例：Shopee 小賣家客服

全部 7 步使用同一個貫穿案例，方便學生看到「上一步的產出就是下一步的輸入」：

| Phase | 關鍵產出 |
|-------|---------|
| 01 | 5 個結構化痛點（手作飾品賣家每天花 2 小時回 LINE 重複問題） |
| 02 | 主題 A「小型電商賣家的重複客服勞動」+ 物理量畫布 |
| 03 | 低端破壞機會：用極簡客服助手服務被 SaaS 過度服務的小賣家 |
| 04 | Strategy Canvas：7 維度 + 差異化摘要 + 不做清單 5 項 |
| 05 | 最小產品：NT$800 一次性，3 小時手工交付 LINE 自動回覆設定 |
| 06 | 4 個 ✅ 強不對稱動機 |
| 07 | GTM 一頁紙 + 72 小時行動計畫（Day 1: 發 LINE 給小美） |

## 核心紀律（貫穿整個流程）

1. **禁止分數** — 不輸出 0–100 分、成功率 X%、可行性 A/B/C
2. **用「問題」不用「點子」** — 你在「釐清問題」不在「驗證點子」
3. **AI 做執行，你做判斷** — AI 負責整理，你負責決定做不做
4. **每一步都有「具體的人」** — 填不出名字 = 先去社群認識人

## 講師備注

### 課前準備

- [ ] 確認 AI 工具可用（ChatGPT / Claude 等）
- [ ] 預先跑過一次 teacher/ 全部 7 步，確認 AI 回覆品質
- [ ] 準備投影設備，方便展示 AI 對話過程
- [ ] 提醒學生準備自己的素材（社群留言、觀察筆記、點子清單）

### 分段教學建議

| 時間 | 建議分段 |
|------|---------|
| 2 小時短課 | Step 01-03（痛點到機會） |
| 3 小時半日課 | Step 01-05（痛點到產品） |
| 5 小時全日課 | Step 01-07（完整流程） |
| 2 天工作坊 | Day 1: Step 01-04 + Day 2: Step 05-07 + 實際發送預售訊息 |

### 學生常見問題

| 問題 | 回應 |
|------|------|
| 「我沒有素材怎麼辦？」 | 花 15 分鐘去 3 個社群抓抱怨貼文 |
| 「AI 的回覆跟老師的不一樣」 | 正常，重點是結構對不對，不是內容一模一樣 |
| 「我填不出具體的人」 | 先暫停，這是最重要的信號：你還不夠接近使用者 |
| 「這個流程跑完就可以開始做了嗎？」 | Step 07 的 72 小時計畫就是你的開始 |
