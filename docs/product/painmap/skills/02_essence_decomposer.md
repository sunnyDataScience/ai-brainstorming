# Phase 2 — 問題本質拆解

> **目標**：把 Phase 1 的多個散亂痛點，聚合成 2–3 個「問題主題」，找出本質問題
> **預估時間**：30–45 分鐘
> **前一步**：`01_pain_collector.md`（需要 ≥ 3 個結構化痛點）
> **下一步**：`03_disruption_scanner.md`

---

## 你需要準備

- Phase 1 產出的結構化痛點清單（≥ 3 個痛點條目）
- 每個痛點都有：誰 × 情境 × 現行解法 × 不滿的點

---

## Prompt（複製貼上到任何 AI 聊天工具）

### Prompt 2A：痛點聚合 + 問題主題提取

```text
你是一位問題本質分析師。我會給你一組結構化的痛點條目，
請幫我做以下三件事：

## 任務一：痛點聚合

把這些痛點按「背後的根本原因」分組。
不是按表面主題分（比如都跟「電商」有關），
而是按深層原因分（比如都是「資訊不對稱」或「重複性手工勞動」）。

每個分組：
- 給一個名稱（≤ 10 個字）
- 說明為什麼這幾個痛點是同一類
- 列出被歸入的痛點編號

## 任務二：問題本質一句話

對每個分組，用一句話說出「問題的本質」。
格式：「[誰] 在 [情境] 面臨的核心問題是 [本質]，
導致他們 [具體的負面結果]」

不要用抽象的詞（「效率低」「體驗差」），
要用可觀察的描述（「每天花 2 小時做重複的事」「每月因此少賺 5000 元」）。

## 任務三：物理量檢驗

對每個問題主題，逐一檢驗四個「物理量」：

| 物理量 | 狀態 | 說明 |
|--------|------|------|
| 具體的人 | ✅ 有 / ⚠️ 模糊 / ❌ 沒有 | 能說出名字嗎？ |
| 正在付錢的痛 | ✅ 有 / ⚠️ 模糊 / ❌ 沒有 | 他們現在有花錢/時間解決嗎？ |
| 手工可交付 | ✅ 有 / ⚠️ 模糊 / ❌ 沒有 | 不寫程式能幫他做一次嗎？ |
| 收款管道 | ✅ 有 / ⚠️ 模糊 / ❌ 沒有 | 有現成的收錢方式嗎？ |

對每個 ⚠️ 或 ❌，給出「你需要做什麼才能補上」的具體行動建議。

## 規則
- 不輸出分數、排名、成功率
- 分組數量 2–5 個（太多 = 沒有真的聚合）
- 如果某些痛點不屬於任何分組，單獨列出並說明原因
- 「問題本質」必須是可被訪談驗證的（我可以拿去問真人「你是不是這樣？」）

我的痛點清單：

{{把 Phase 1 產出的所有痛點條目貼在這裡}}
```

### Prompt 2B：深挖單一問題主題

如果 Prompt 2A 找出的某個問題主題讓你特別感興趣，用這個 prompt 深入挖掘：

```text
你是一位第一性原理分析師。我有一個初步的「問題主題」，
我想更深入理解它的本質。

我的問題主題：
{{貼上 Prompt 2A 產出的某個問題主題的一句話描述}}

包含的痛點：
{{貼上歸入這個主題的痛點條目}}

請幫我做以下分析：

### 1. 五個為什麼（5 Whys）
從表面問題往下挖 5 層，找到根本原因。
每一層都要回答「為什麼會這樣？」

### 2. JTBD（Jobs to be Done）完整版
用完整格式描述這個問題：
「當 [具體的人] 在 [情境] 時，
 他想要 [功能性目標]，
 好讓他可以 [情感性目標]，
 但現在 [阻礙/現行解法的問題]，
 導致他 [具體負面結果]。」

### 3. 替代方案深度分析
目前的替代方案（使用者現在怎麼解決）是什麼？
- 名稱：
- 成本（錢 + 時間）：
- 使用者最不滿的 3 件事：
- 使用者最離不開的 1 件事（即使不滿也不會換的原因）：

### 4. 問題邊界
- 這個問題在什麼條件下存在？（行業、規模、地區？）
- 什麼時候這個問題不存在？（哪些人不會遇到？）
- 這個問題在變嚴重還是變輕？（趨勢）

### 規則
- 不打分數
- 如果資訊不足以回答某一項，標注「需要訪談 [誰] 才能確認」
- 替代方案分析中的「使用者最不滿的 3 件事」必須是可觀察的行為，
  不是推測的情緒
```

## AI 回覆後你要做的事

### 1. 驗證聚合是否合理

AI 把你的痛點分成了幾組。對每組問自己：
- 「這幾個痛點真的是同一個根本問題嗎？」
- 如果覺得不對 → 告訴 AI 你的想法，讓它重新分組

### 2. 選出你最想深入的 1–2 個問題主題

選擇標準（不是分數，是直覺 + 事實）：
- **哪個主題我認識最多有這個問題的人？**（物理量 #1）
- **哪個主題的替代方案最爛？**（物理量 #2 的機會）
- **哪個主題我今晚就能幫一個人？**（物理量 #3）

### 3. 對選出的主題，手動填寫物理量畫布

```markdown
## 問題物理量畫布

### 物理量 #1：具體的人
我現在能聯絡的、有這個問題的人：
1. 姓名：___  關係：___  聯絡方式：___
2. 姓名：___  關係：___  聯絡方式：___
3. 姓名：___  關係：___  聯絡方式：___
（填不出 3 個 → 先去社群認識人）

### 物理量 #2：正在付錢的痛
這 3 個人現在怎麼解決？花多少錢 / 時間？
1. 人 A 用：___，每月花：___
2. 人 B 用：___，每月花：___
3. 人 C 用：___，每月花：___
（如果答案是「他們沒在解決」→ 可能不是真痛點）

### 物理量 #3：手工可交付
如果我今晚幫人 A 做一次，步驟是：
1. ___
2. ___
3. ___
（如果第一步就是「寫程式」→ 重想，手工版不含技術）

### 物理量 #4：收款管道
我手上可用的收款方式：___
（沒有 → 先去開一個 Lemon Squeezy / Stripe / TapPay / 銀行轉帳）
```

**這一步 AI 不能替你做。** 物理量畫布必須是你自己填的真實資訊。

## 常見卡關與解法

### 卡關 A：「AI 分出的組看起來都差不多」

**原因**：你的痛點可能真的是同一個問題的不同面向。
**解法**：合併成 1 個大主題，然後用 Prompt 2B 深挖。一個清晰的大問題比 5 個模糊的小問題有用。

### 卡關 B：「每個主題看起來都想做，選不出來」

**原因**：你在用「哪個比較好」的思維，但這不是選擇題。
**解法**：用物理量選。**哪個主題你認識最多有這個問題的人？** 就選那個。人最多 = 訪談最容易做 = 驗證最快。

### 卡關 C：「物理量畫布填不滿」

**原因**：你對這個問題的理解還不夠具體。
**解法**：這正常。標記哪些物理量缺失，帶著這個「缺口」繼續往前。Phase 3–7 的過程會幫你逐步補上。但如果「具體的人」完全填不出來 → 需要先暫停去找社群。
