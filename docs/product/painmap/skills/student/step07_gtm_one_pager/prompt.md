## Prompt 7B：72 小時行動計畫紅隊檢查（拿到 7A 產出後接著跑）

```text
你是一位嚴格的執行力教練。我剛寫好一份 GTM 一頁紙和 72 小時行動計畫。
請用以下 5 個檢查點挑戰我，確保我明天真的能開始做：

{{把 Prompt 7A 的完整 AI 回覆貼在這裡}}

### 檢查 1：「前 10 個客戶」是否真的具體？
- 有沒有任何一個是抽象描述（「目標客群」「潛在使用者」）？
- 至少 3 個有聯絡方式或具體管道嗎？
- 如果我今天就要聯絡，我知道怎麼找到他們嗎？

### 檢查 2：Day 1 的第一個動作是否「5 分鐘內能開始」？
- 不需要先做任何準備嗎？
- 不需要先「研究」或「思考」嗎？
- 是一個物理動作（發訊息、打電話、寫 email）而非認知動作嗎？

### 檢查 3：72 小時計畫有沒有「代理動作」？
代理動作 = 看起來像進展但不直接服務「收到第一塊錢」的動作。
- 做 logo ← 代理動作
- 選技術棧 ← 代理動作
- 寫 landing page ← 可能是代理動作（取決於是否為了收款）
- 發訊息給真人 ← 真實動作
- 打電話 ← 真實動作
- 手工交付一次 ← 真實動作

標出所有代理動作，建議替換成真實動作。

### 檢查 4：成功 / 放棄標準是否明確？
- 「可以繼續」的標準是否可觀察（不是「感覺不錯」）？
- 「放棄」的標準是否足夠具體（不是「如果不行就算了」）？
- 72 小時後，我能不能看著這個標準明確地說「達標」或「沒達標」？

### 檢查 5：有沒有遺漏「收款管道」？
- 72 小時計畫中，有沒有一個步驟是「確保收款連結可用」？
- 如果有人回「好」，我 5 分鐘內能發出付款連結嗎？
- 如果收款工具還沒設定 → Day 1 必須加入

### 規則
- 不安慰我
- 發現問題直說，並給具體的替換建議
- 如果我的計畫已經很好，說「沒有重大問題，去做」，不要硬找問題
```

## 常見陷阱

### 陷阱 1：「前 10 個客戶」寫的是族群，不是人
錯誤：「4-7. Shopee 賣家社群的用戶」
正確：「4-6. Facebook『蝦皮賣家交流討論區』：發研究型貼文，私訊回覆的成員」

### 陷阱 2：Day 1 第一個動作需要「先準備一下」
錯誤：「建立一個 Google 表單收集問卷」（需要 30 分鐘準備）
正確：「打開 LINE，發訊息給 ___（名字）」（現在就能做）

### 陷阱 3：計畫裡有隱藏的代理動作
常見偽裝：「研究競品」「整理一份說明文件」「做一個簡單的網站」
測試方法：問自己「這個動作完成後，有沒有讓我更靠近收到錢？」

### 陷阱 4：忘記設定收款連結
後果：有人說「好，怎麼付？」，你說「等我先去開一個帳號……」→ 對方冷掉

### 陷阱 5：放棄標準太模糊
錯誤：「如果沒有人買，就重新想想」
正確：「發了 10 封訊息，0 人回應 → 回 Phase 5 改預售訊息；
有回應但 0 人付款，且都說嫌貴 → 試試降 50% 定價再跑一輪」

## AI 給出 7A 後，你要做的事

### 1. 立刻跑 Prompt 7B 紅隊檢查

把 7A 的完整輸出貼入 7B，讓 AI 自己挑戰自己的計畫。

### 2. 根據紅隊結果調整

常見需要調整的地方：
- Day 1 第一個動作不夠具體 → 改成「打開 ___，發訊息給（名字）」
- 缺收款連結 → Day 1 加入「花 30 分鐘開好 LINE Pay / 歐付寶」
- 有代理動作 → 替換成「聯絡真人」的動作
- 放棄標準太籠統 → 拆成 A / B / C 三個診斷情境

### 3. 把 GTM 一頁紙存起來

真的存成一張截圖放在手機桌面，或打印出來放在桌上。

為什麼？因為接下來 72 小時你會想「要不要先做個 logo」「要不要先建個網站」
「要不要先想好名字」。每次有這個念頭 → 看一眼你的一頁紙 → 回到行動清單。

### 4. 設定 72 小時倒數

用手機設一個 72 小時倒數。72 小時後，你會知道兩件事之一：
- 有人付錢了 → 你找到了一個真實存在的生意的種子
- 沒人付錢 → 你知道「為什麼沒人付」，這比「不知道做什麼」好 100 倍

### 5. 現在就發出第一封訊息

不是「等準備好」。不是「等完美」。不是「等到明天」。

你在 Phase 5 已經寫好了預售訊息草稿。改一改最後一句話。然後按發送。

## 流程結束後

### 如果 72 小時後有人付錢了

你找到了一個真實存在的生意的種子。下一步：

1. **交付第一單**（做到最好，這是你的口碑起點）
2. **記錄整個過程花了多少時間**（更新你的 Unit Economics 真實數據）
3. **請客戶寫 2 句話回饋**（用來向下一個客戶展示）
4. **找第 2、第 3 個客戶**（用同樣的方式）
5. 如果你想把手工流程產品化 → 回到技術，開始寫程式

### 如果 72 小時後沒人付錢

不要馬上放棄，也不要馬上再來一輪。

**第一步：寫一份覆盤（30 分鐘）**
- 發了幾封訊息？幾個人回覆？幾個人說「有興趣」？幾個人付錢？
- 拒絕的人的理由是什麼？
- 我的 Unit Economics 哪個假設最可能是錯的？

**第二步：用覆盤結果診斷**

| 觀察 | 診斷 | 下一步 |
| :--- | :--- | :--- |
| 沒人回覆 | 信任不夠 / 訊息寫得像廣告 | 回 Phase 5 改預售訊息 |
| 有人聊但沒人付 | 痛點不夠強 or 解法不對 | 回 Phase 2 重新看問題本質 |
| 有人想付但嫌貴 | 定價錯 | 降 50% 再試一次 |
| 有人付了但覺得沒用 | 手工交付不到位 | 回 Phase 5 改交付步驟 |

**第三步：休息 48 小時**，然後帶著新發現重跑你需要的 Phase。

---

## 最後提醒

這 7 個 Phase 的全部價值，都在最後這一步：你發出了一封訊息，有人付了錢（或沒有）。

如果你跑完 7 個 Phase 但沒有發出任何一封訊息，你做的就是另一種形式的代理動作。

去做吧。
