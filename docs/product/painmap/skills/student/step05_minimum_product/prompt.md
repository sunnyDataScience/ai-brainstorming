# Step 05 最小破壞性產品設計 — 學生演練版 Prompt

> **版本**：學生演練版 — Phase 5
> **對應技能**：`05_minimum_product.md` — Prompt 5A + Prompt 5B
> **前置條件**：完成 Phase 4（需要 Strategy Canvas、差異化摘要、不做清單）
> **預估時間**：30–45 分鐘

---

## 核心觀念（開始前讀一遍）

**最小破壞性產品 ≠ MVP。**

MVP 通常是「功能最少的軟體」。我們要做的更極端：

> **一個不需要寫任何程式、你今晚就能用手做一次、然後收到錢的東西。**

為什麼從手工開始？

| 直接寫程式 | 先手工交付 |
|-----------|-----------|
| 需要幾週才能開始 | 今晚就能開始 |
| 寫完才知道有沒有人要 | 交付前就能知道 |
| 技術風險 + 市場風險同時存在 | 先驗證市場，再考慮技術 |
| 沒做過就不知道要自動化什麼 | 手工過程會告訴你「真正需要自動化什麼」 |

**黃金法則**：如果手工做都沒人買，寫成程式也不會有人買。

## Prompt 5A：最小產品設計

### 使用說明

1. 把你在 Phase 2–4 整理出來的資訊填入以下 `{{ }}` 欄位
2. 完整複製整段 prompt，貼入 AI 聊天工具（ChatGPT / Claude / Gemini 皆可）
3. 收到回覆後，對照下方「收到回覆後的檢查點」逐一確認

### 填入你的資訊前，先準備好：

- [ ] Phase 2 的問題本質（一句話）
- [ ] Phase 2 的具體目標客群描述
- [ ] Phase 4 的差異化定位摘要
- [ ] Phase 4 的 Strategy Canvas 拉高維度清單
- [ ] Phase 4 的不做清單

---

### Prompt 5A 本文（填空後複製貼上）

```text
你是一位最小產品設計師。我的目標是設計一個「不寫程式、
今晚就能手工做一次、然後收到錢」的最小產品。

## 我的資訊

**問題本質**：{{ 填入 Phase 2 的一句話問題本質 }}
**目標族群**：{{ 填入具體描述，包含月營收/規模/使用工具/痛點情境 }}
**差異化定位**：{{ 填入 Phase 4 的差異化摘要，說明你用什麼方式服務哪個被過度服務的族群 }}
**只做**：
- {{ 拉高維度 1 }}
- {{ 拉高維度 2 }}
- {{ 拉高維度 3（如果有） }}
**不做**：
- {{ 不做項目 1 }}
- {{ 不做項目 2 }}
- {{ 不做項目 3 }}
- {{ 不做項目 4（如果有） }}

---

請幫我設計：

### 1. 核心價值主張（一句話）

格式：「幫 [誰] 在 [情境] 用 [更___的方式] 解決 [什麼問題]」
要求：一個不知道你是誰的人，讀完這句話就知道「我要不要繼續看」

### 2. 只做三件事

列出這個產品只做的 3 件事。每件事用一句話描述。
規則：
- 必須對應 Strategy Canvas 中你「拉高」的維度
- 每件事都是「客戶可感知的交付」，不是「技術實作」
  （「幫你自動回覆常見問題」✓ 「建立 NLP 意圖辨識模型」✗）
- 如果你想加第 4 件 → 先砍一件

### 3. 不做清單（≥ 3 項）

從 Strategy Canvas 的「刻意不做」延伸，列出至少 3 件不做的事。
每件附理由：「不做 ___，因為 ___」

### 4. 手工交付版本（最重要的一段）

如果今晚就要幫 1 個人做一次，具體步驟是什麼？

規則：
- 步驟中不能出現「寫程式」「做 App」「建系統」
- 可以用的工具：Google Doc / Notion / Excel / Email /
  LINE / 電話 / Canva / 手機
- 必須能在 3 小時內做完一次
- 每個步驟標注預估時間

### 5. 定價設計

- 建議售價：NT$ ___
- 定價邏輯：這個價格 vs 他們現行解法的成本
- 收費方式：一次性 / 月費 / 按次計費
- 退費承諾：不滿意如何退費

### 6. Unit Economics 框算

- 每單收入：NT$ ___
- 每單成本：NT$ ___（含你的時間 × 你的時薪估計）
- 毛利：NT$ ___
- 毛利 > 0？___
- 關鍵假設：___

如果毛利 ≤ 0，指出問題並建議調整。

### 規則
- 不打分數
- 手工交付版本是整個輸出最重要的部分，必須具體到「我照著做就能做」
- 如果某個環節「手工做不到」→ 建議砍範圍，而非建議寫程式
```

---

### 收到 5A 回覆後：自我檢查點

**核心價值主張：**
- [ ] 有「誰」（具體的人，不是「所有人」）
- [ ] 有「情境」（他在什麼狀況下用）
- [ ] 有「結果」（幫他達到什麼）

**只做三件事：**
- [ ] 每件事我讀完都知道「客戶拿到什麼」
- [ ] 沒有一件事需要「寫程式」才能交付
- [ ] 三件事加起來對應你 Strategy Canvas 的拉高維度

**手工交付步驟（最關鍵的檢查）：**
- [ ] 所有步驟加起來 ≤ 3 小時
- [ ] 每個步驟有具體的「用什麼工具做什麼」
- [ ] 沒有步驟出現「系統」「App」「串接」「寫程式」等字眼
- [ ] 步驟做完後，客戶手上有「拿得到的東西」（文件、設定、影片……）

**Unit Economics：**
- [ ] 知道第幾單開始毛利轉正
- [ ] 知道讓成本下降的關鍵是什麼（標準化？模板複用？）

### Prompt 5B 本文（填空後複製貼上）

```text
你是一位台灣 solo founder 的預售文案寫手。

我要寫一封訊息給我認識的人，告訴他「我可以幫你做一件事，
你願意付錢嗎？」

## 我的資訊

**收件人**：{{ 名字、身份、你們的關係，以及你知道的他的具體痛點情境 }}
**他的痛點**：{{ 他遇到的具體問題，盡量用他自己說過的話 }}
**我要提供的手工交付物**：
{{ 把 5A 的手工交付步驟用 1-5 條列出來 }}
**價格**：NT${{ 金額 }}（{{ 一次性 / 月費 }}）
**交付時間**：{{ 幾天內完成，以及是否有追蹤 }}

---

請幫我寫一封 LINE 訊息或 email（≤ 150 字），結構：

1. **為什麼找他**（一句話，引用他的具體情境）
2. **我要幫他做什麼**（一句話，具體到他能想像）
3. **價格 + 交付時間 + 退費承諾**
4. **行動呼籲**（回「好」就開始）

### 規則
- 不用「革命性」「顛覆」「AI 驅動」等形容詞
- 不提「我的產品」「我的 app」（你還沒有產品，你在手工做）
- 語氣像朋友幫忙，不像銷售
- 台灣口語，不過度正式也不過度網路化
- 包含退費承諾（「不滿意全退」）
```

---

### 收到 5B 回覆後：自我檢查點

- [ ] 訊息開頭有引用他的具體情境（不是「你好，我在做一個服務……」）
- [ ] 「我要幫你做什麼」是他能想像的畫面，不是功能列表
- [ ] 有明確的一次性 / 月費說明
- [ ] 有退費承諾
- [ ] 行動呼籲只有一個（回「好」、點連結……），不是「有空的話聯絡我」
- [ ] 請旁邊同學聽你讀這封訊息——他說聽起來像朋友還是業務？

## 常見卡關與解法

### 卡關 A：「手工做不到，一定要寫程式」

99% 的情況是範圍太大。試著問自己：

> 「整個流程中，哪個步驟如果由我手動做、成果其實一樣？」
> 「客戶在意的是結果還是過程？」

**砍範圍的練習：**

把你的手工交付步驟讀一遍，對每個步驟問：「如果這個步驟我改用 Google Doc + 電話完成，客戶的體驗差多少？」

- 差很多 → 這個步驟可能是核心，保留但想手工替代方案
- 差不多 → 就用 Google Doc + 電話，不需要程式

### 卡關 B：「定價不知道怎麼定」

**最簡單的方法**：他現在的替代方案花多少錢（或多少時間）？你的定價 = 替代方案成本的 30–50%。

- 他現在花 NT$2,000/月 → 你收 NT$600–1,000
- 他現在花 2 小時/天 → 你幫他省 1 小時 → 他的時間值多少？

**第一次定價一定是錯的。沒關係。先收到錢，再調整。**

### 卡關 C：「不做清單太短，砍不下去」

對每個「要做」的功能問：「如果不做這個，客戶還是會付錢嗎？」

- 會 → 砍掉
- 不會 → 保留
- 不確定 → 先砍，等客戶要求再加回來

### 卡關 D：「Unit Economics 是負的，是不是不能做？」

前 3–5 單的毛利是負的，這是正常的「學習成本」。
重要的問題不是「第 1 單賺多少」，而是：

> 「第 5 單起，我能把時間成本壓下來嗎？靠什麼壓？」

如果你想不出答案 → 問 AI：「我目前的手工交付步驟哪些可以在第 3 單後標準化？」
