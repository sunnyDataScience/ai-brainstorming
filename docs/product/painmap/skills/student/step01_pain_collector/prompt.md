# Step 01 痛點採集 — 學生演練版

> **版本**：學生演練版 — Phase 1 痛點採集
> **目標**：把你手邊的散亂素材（留言、對話、觀察筆記、腦中想法），
>            轉化成有結構的痛點條目
> **預估時間**：30–60 分鐘
> **前一步**：無（這是起點）
> **下一步**：`02_essence_decomposer.md`

---

## 前置準備

在開始之前，請先準備至少**一種**素材：

| 素材類型 | 範例 | 適合誰 |
| :--- | :--- | :--- |
| **社群留言 / 對話** | Facebook 社團、LINE 群組、Discord、PTT 的對話截圖或複製文字 | 有在參與社群的人 |
| **客戶 / 受眾回饋** | YouTube 留言、IG 私訊、客服對話、email 回信 | 有受眾的創作者、賣家 |
| **你的觀察筆記** | 你平時記下的「這件事好煩」「為什麼沒人做這個」的筆記 | 有行業經驗的人 |
| **腦中的想法** | 「我覺得 XX 行業有個問題……」的口述描述 | 任何人 |
| **多個 idea** | 你記錄過的點子清單（idea.md、Notion、備忘錄） | 有很多想法但不知道選哪個的人 |

**還沒有素材？**
→ 去你最常逛的 3 個社群（Facebook 社團 / Discord / PTT），花 15 分鐘把「有人在抱怨什麼」的貼文複製下來。這就是你的素材。

---

## Prompt 選擇指引

| 我的狀況 | 用哪個 Prompt |
| :--- | :--- |
| 我有留言、對話、觀察筆記等文字素材 | **Prompt 1A** |
| 我有一堆點子但不知道哪個是真痛點 | **Prompt 1B** |
| 我腦中有模糊想法但說不清楚 | **Prompt 1C（蘇格拉底對話）** |

## Prompt 1A：從散亂素材提取痛點

> **適用時機**：你有留言截圖、對話紀錄、觀察筆記等文字素材

複製以下整段文字，貼入 AI 聊天工具，並將 `{{在這裡貼上你的素材}}` 替換為你的實際素材。

```text
你是一位痛點結構化專家。我會給你一段原始素材（可能是社群留言、
客戶對話、我的觀察筆記、或腦中想法的口述）。

請幫我從中提取痛點，每個痛點用以下格式輸出：

---
### 痛點 #N
- **誰**：遇到這個問題的是什麼樣的人？（盡量具體：職業、情境、身份）
- **情境**：他在什麼時候、什麼情況下遇到這個問題？
- **現行解法**：他現在怎麼處理？用什麼工具 / 方法 / 找什麼人？
- **不滿的點**：現行解法讓他最不爽的是什麼？
- **一句話痛點**：用一句話總結（格式：「當 [誰] 在 [情境] 時，他想要 [什麼]，
  但現在只能 [現行解法]，讓他覺得 [不滿]」）
---

規則：
1. 每個痛點必須有「具體的人」，不能是抽象的群體（「年輕人」不行，
   「在 Shopee 賣飾品的 30 歲媽媽」可以）
2. 「現行解法」必須有名字（是 Excel？是某個 App？是找外包？是忍受？）
   如果素材中沒提到，標注「需要訪談確認」
3. 不要輸出任何分數、評分、可行性判斷
4. 如果素材中看不出某個欄位，標注「資訊不足」而非編造
5. 一段素材可能包含多個痛點，全部提取出來

我的素材如下：

{{在這裡貼上你的素材}}
```

## Prompt 1B：從多個 idea 提取痛點

> **適用時機**：你有的是一堆點子，不確定哪些背後有真實痛點

複製以下整段文字，貼入 AI 聊天工具，並將 `{{點子名稱}}` 與 `{{在這裡列出你的點子，一個一行}}` 替換為你的實際內容。

```text
你是一位痛點結構化專家。我有一份點子清單，但我不確定哪些是真的痛點、
哪些只是我的想像。

請幫我逐一檢視每個點子，用以下格式輸出：

---
### 點子 #N：{{點子名稱}}

**痛點拆解**：
- **誰**：這個點子要服務的是什麼人？（具體描述）
- **情境**：他們在什麼情況下會遇到這個問題？
- **現行解法**：他們現在怎麼處理？
- **不滿的點**：現行解法差在哪？
- **一句話痛點**：「當 [誰] 在 [情境] 時……」

**物理量檢驗**：
- ☐ 具體的人：我能列出 3 個認識的、有這個問題的人嗎？
- ☐ 正在付錢的痛：他們現在有花錢 / 花時間解決嗎？
- ☐ 手工可交付：不寫程式，我今晚能幫一個人做一次嗎？
- ☐ 收款管道：我有現成的收款方式嗎？

**缺什麼**：這個點子目前最缺的物理量是什麼？要補上需要做什麼？
---

規則：
1. 不打分數、不排名、不預測成功率
2. 「物理量檢驗」的 4 個勾選框由我自己填，你只提供拆解讓我判斷
3. 如果一個點子看起來「不像真的痛點」（沒有人在付錢解決），直說
4. 如果資訊不足以判斷，標注「需要找 3 個真人聊聊才能確認」

我的點子清單：

{{在這裡列出你的點子，一個一行}}
```

## Prompt 1C：蘇格拉底對話模式（腦中有想法但說不清楚）

> **適用時機**：你腦中有模糊的方向，但組織不起來，需要 AI 用提問引導你

複製以下整段文字，貼入 AI 聊天工具，並將 `{{你的領域 / 主題}}` 替換為你關心的主題。

```text
你是一位蘇格拉底式的痛點挖掘教練。你的工作是透過提問，幫我把腦中
模糊的想法結構化成清晰的痛點定義。

規則：
1. 每次只問我一個問題
2. 問題必須具體，不能是「你想做什麼？」這種開放題
3. 根據我的回答追問，直到釐清以下四個「物理量」：
   - 具體的人（有名字的、我認識的人）
   - 正在付錢的痛（他們現在花錢 / 時間 / 精力在解決什麼）
   - 手工可交付（我不寫程式，今晚能幫他做一次什麼）
   - 收款管道（我用什麼收錢）
4. 每釐清一個物理量，幫我小結一次
5. 四個都釐清後，幫我產出完整的「一句話痛點」和「物理量畫布」
6. 如果我的回答太抽象（「我想幫創業者」），直接指出並要求更具體
7. 不給分數、不判斷好壞、不給建議「你應該做 X」

我們開始吧。我目前腦中有一個模糊的想法，跟 {{你的領域 / 主題}} 有關。
```

## AI 回覆後你要做的事

**這一步 AI 不能替你做。**

拿到 AI 的結構化痛點後，對每一個痛點問自己：

1. **「誰」那一欄，我能說出一個真人的名字嗎？**
   - 能 → 在旁邊寫下名字和聯絡方式
   - 不能 → 這個痛點還不夠具體，標記「需要找人」

2. **「現行解法」那一欄，是真的還是 AI 猜的？**
   - 是我自己知道的 / 素材裡說的 → OK
   - 是 AI 推測的 → 標記「需要訪談確認」

3. **如果我今晚要幫這個人做一次，我會做什麼？**
   - 能具體說出步驟 → 寫下來
   - 說不出來 → 這個痛點可能太大，需要再拆

## 檢查點

完成 Phase 1 後，對照以下清單：

- [ ] 我有 ≥ 3 個結構化的痛點條目
- [ ] 每個痛點都有「誰 × 情境 × 現行解法 × 不滿的點」
- [ ] 至少 1 個痛點，我能說出一個認識的真人的名字
- [ ] 至少 1 個痛點的「現行解法」是確認的（不是 AI 猜的）
- [ ] 沒有任何分數或評分出現在我的筆記中

**全部打勾** → 進入 Phase 2
**有打不了勾的** → 看下面「常見卡關與解法」

## 常見卡關與解法

### 卡關 A：「我貼了素材但 AI 只提取出 1 個痛點」

**原因**：素材太短或太籠統。
**解法**：補更多素材。去你的社群再抓 10 則留言，或把你自己的觀察用 300 字以上描述出來。

### 卡關 B：「AI 提取的痛點看起來都很抽象」

**原因**：你的素材本身就是抽象的（「大家都覺得 XX 很煩」）。
**解法**：切換到 Prompt 1C（蘇格拉底對話），讓 AI 用提問把你逼到具體。

### 卡關 C：「我填不出任何具體的人的名字」

**原因**：你還不屬於任何跟這個痛點相關的社群。
**解法**：先暫停這個流程。花 1–2 週去加入相關社群、幫忙回答問題、認識人。**沒有具體的人 = 還不是做產品的時候。** 這不是壞事，這是你現在最需要做的事。

### 卡關 D：「我有太多痛點（> 10 個），不知道怎麼收斂」

**原因**：這是好事！
**解法**：全部保留，帶進 Phase 2。Phase 2 的工作就是幫你從一堆痛點中找出「問題主題」。
