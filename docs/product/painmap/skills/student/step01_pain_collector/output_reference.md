> **版本**：學生演練版 — 產出格式參考

> **重要提醒**：你的產出應該有類似的結構，但內容完全是你自己的素材與痛點。

## 填寫範例（蝦皮小賣家客服痛點）

> 這是老師課堂示範用的素材所產出的範例。
> 參考它的「顆粒度」和「具體程度」，不要複製內容。

```markdown
# 我的痛點採集 — 2026-04-09

## 素材來源
- Facebook 電商社團留言（1 則）
- 朋友小美的抱怨（口頭，我整理成筆記）
- PTT 電商版貼文（1 則）

## 結構化痛點

### 痛點 #1
- **誰**：在 Shopee 賣手作飾品的 30 歲媽媽（像我朋友小美）
- **情境**：每天晚上 9 點小孩睡了之後要花 2 小時回 LINE 客服訊息
- **現行解法**：手動逐則回覆 + 用 Excel 記訂單（需要訪談確認是否有用平台快速回覆）
- **不滿的點**：重複回答一樣的問題（尺寸、運費、材質）；不敢不回怕丟客人
- **一句話痛點**：「當手作飾品賣家在晚上處理客服時，她想要快速回覆
  常見問題，但現在只能逐則手打 LINE，讓她覺得浪費時間且不敢停下來」
- **我認識的人**：小美（LINE 聯絡方式有）
- **物理量**：☑ 具體的人　☑ 正在付錢的痛（花時間）　☐ 手工可交付（需想）　☑ 收款管道

### 痛點 #2
- **誰**：在 PTT 電商版發文、有共鳴的電商小賣家（身份待確認）
- **情境**：整體日常營運中，感受到重複回訊息是最浪費時間的環節
- **現行解法**：資訊不足（素材只陳述感受，未提工具）
- **不滿的點**：把「重複回訊息」視為最大的時間黑洞
- **一句話痛點**：「當電商小賣家在經營店鋪時，他想要把時間花在更有價值的地方，
  但現在每天都被重複客服佔據，讓他覺得時間被嚴重浪費」
- **我認識的人**：需要找人（可去 PTT 原文留言詢問）
- **物理量**：☐ 具體的人　☑ 正在付錢的痛（花時間）　☐ 手工可交付　☑ 收款管道

### 痛點 #3
- **誰**：兼顧家庭或正職工作的斜槓小賣家（從「晚上才有空回」推論）
- **情境**：只有下班後或家務結束後的零碎時間可用於回覆，時間壓力大
- **現行解法**：需要訪談確認（AI 推論為純手動回覆，未經確認）
- **不滿的點**：私人時間被客服工作佔據，生活與事業難以平衡
- **一句話痛點**：「當斜槓小賣家在晚間唯一的空閒時段處理客服時，她想要
  高效率解決完，但現在只能花 2 小時手動回覆，讓她覺得沒辦法好好休息」
- **我認識的人**：需要找人（這個推論需要訪談確認）
- **物理量**：☐ 具體的人　☑ 正在付錢的痛（花時間）　☐ 手工可交付　☑ 收款管道

## 待確認
- 痛點 #1 的「現行解法」需確認小美是否有用 Shopee 平台自動回覆功能
- 痛點 #1 的「手工可交付」需想清楚：我今晚能幫小美做什麼？
- 痛點 #2 的「具體的人」需要去找有這個問題的真實電商賣家
- 痛點 #3 整體需要訪談確認（AI 推論，非素材明確陳述）
```

## 品質對照表

對照你的產出，確認是否達到以下標準：

| 品質指標 | 範例做到了嗎？ | 說明 |
| :--- | :--- | :--- |
| 「誰」有職業 / 情境描述，不是抽象群體 | 有（「蝦皮手作飾品賣家、30 歲媽媽」） | 越具體越好 |
| 「現行解法」有名字或標注待確認 | 有（LINE 手動回覆 / 標注訪談確認） | 不能空白或寫「不知道」 |
| 「一句話痛點」遵循固定句式 | 有 | 句式要有「誰、情境、想要、現行解法、不滿」 |
| 「我認識的人」有具體線索 | 部分有（小美） | 至少 1 個痛點要有 |
| 「待確認」有列出 | 有 | AI 猜的都要標出來 |
| 沒有分數或評分 | 沒有 | 嚴格禁止 |
