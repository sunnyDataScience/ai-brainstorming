> **目標**：把 Strategy Canvas 轉化成一個「最小能賣的東西」——今晚就能開始手工交付
> **預估時間**：30–45 分鐘
> **前一步**：`04_strategy_canvas.md`（需要策略圖 + 差異化摘要 + 不做清單）
> **下一步**：`06_asymmetry_check.md`

## 核心觀念（1 分鐘讀完）

**最小破壞性產品 ≠ MVP。**

MVP 通常的意思是「功能最少的軟體」。但我們要做的更極端：

> **一個不需要寫任何程式、你今晚就能用手做一次、然後收到錢的東西。**

為什麼？
- 寫程式需要時間 → 時間 = 延遲驗證 = 風險
- 如果手工做都沒人買 → 寫成程式也不會有人買
- 手工做的過程會教你「真正需要自動化什麼」

## Prompt（複製貼上到任何 AI 聊天工具）

### Prompt 5A：最小產品設計

```text
你是一位最小產品設計師。我的目標是設計一個「不寫程式、
今晚就能手工做一次、然後收到錢」的最小產品。

## 我的資訊

**問題本質**：{{Phase 2 的一句話}}
**目標族群**：{{具體描述}}
**差異化定位**：{{Phase 4 的差異化摘要}}
**只做**：{{Strategy Canvas 中拉高的維度}}
**不做**：{{不做清單}}

---

請幫我設計：

### 1. 核心價值主張（一句話）

格式：「幫 [誰] 在 [情境] 用 [更___的方式] 解決 [什麼問題]」
要求：一個不知道你是誰的人，讀完這句話就知道「我要不要繼續看」

### 2. 只做三件事

列出這個產品只做的 3 件事。每件事用一句話描述。
規則：
- 必須對應 Strategy Canvas 中你「拉高」的維度
- 每件事都是「客戶可感知的交付」，不是「技術實作」
  （「幫你自動回覆常見問題」✓ 「建立 NLP 意圖辨識模型」✗）
- 如果你想加第 4 件 → 先砍一件

### 3. 不做清單（≥ 3 項）

從 Strategy Canvas 的「刻意不做」延伸，列出至少 3 件不做的事。
每件附理由：「不做 ___，因為 ___」

### 4. 手工交付版本（最重要的一段）

如果今晚就要幫 1 個人做一次，具體步驟是什麼？

規則：
- 步驟中不能出現「寫程式」「做 App」「建系統」
- 可以用的工具：Google Doc / Notion / Excel / Email /
  LINE / 電話 / Canva / 手機
- 必須能在 3 小時內做完一次
- 每個步驟標注預估時間

### 5. 定價設計

- 建議售價：NT$ ___
- 定價邏輯：這個價格 vs 他們現行解法的成本
- 收費方式：一次性 / 月費 / 按次計費
- 退費承諾：不滿意如何退費

### 6. Unit Economics 框算

- 每單收入：NT$ ___
- 每單成本：NT$ ___（含你的時間 × 你的時薪估計）
- 毛利：NT$ ___
- 毛利 > 0？___
- 關鍵假設：___

如果毛利 ≤ 0，指出問題並建議調整。

### 規則
- 不打分數
- 手工交付版本是整個輸出最重要的部分，必須具體到「我照著做就能做」
- 如果某個環節「手工做不到」→ 建議砍範圍，而非建議寫程式
```

## AI 回覆後你要做的事

### 1. 用手工交付版本做一次「桌面演練」

不要真的找客戶，先自己在腦中 / 紙上跑一次：

- 步驟 1 做得到嗎？需要什麼工具？
- 步驟 2 呢？會卡在哪裡？
- 整個流程真的 3 小時內能做完嗎？
- 做完後客戶拿到的是什麼？他會覺得「值 NT$X」嗎？

如果任何步驟做不到 → 砍範圍，直到做得到。

### 2. 檢查 Unit Economics

如果你的時薪估計是 NT$500/hr，做一次要 3 小時，成本就是 NT$1500。
如果收費 NT$500 → 你每做一單虧 NT$1000。
這不代表不能做 → 前幾單虧錢是正常的（你在學習和驗證）。
但你需要知道「未來自動化後成本會降到多少」才有辦法持續。

### 3. 挑出前 3 個客戶

從 Phase 2 的物理量畫布中，選出最可能回「好」的 3 個人。
用 Prompt 5B 為每個人寫一封訊息草稿。

**先不要發。** Phase 6 還要做一次檢查。

## 常見卡關與解法

### 卡關 A：「手工做不到，一定要寫程式」

**99% 的情況是範圍太大。** 問自己：
- 「整個流程中，哪個步驟如果由我手動做、成果其實一樣？」
- 「客戶在意的是結果還是過程？」

範例：
- 「自動回覆客服」→ 手工版：你幫他寫好 10 個常見問答的 LINE 快捷回覆腳本，教他怎麼設定
- 「AI 分析留言」→ 手工版：你花 1 小時讀他的 50 則留言，整理出 Top 3 需求寄給他

### 卡關 B：「定價不知道怎麼定」

**最簡單的方法**：他現在的替代方案花多少錢（或多少時間）？你的定價 = 替代方案成本的 30–50%。
- 他現在花 NT$2000/月 → 你收 NT$600–1000
- 他現在花 2hr/天 → 你幫他省 1hr → 他的時間值多少？

**第一次定價一定是錯的。** 沒關係。先收到錢，再調價格。

### 卡關 C：「不做清單太短，砍不下去」

**對每個「要做」的功能問**：「如果不做這個，客戶還是會付錢嗎？」
- 會 → 砍掉
- 不會 → 保留
- 不確定 → 先砍，等客戶要求再加回來

## 產出範本

```markdown
# 最小產品設計 — {{日期}}

## 核心價值主張
幫 Shopee 小賣家用 5 分鐘設定好常見問答自動回覆，
每天省 1 小時重複客服時間

## 只做三件事
1. 幫你整理 10 個最常被問的問題 + 寫好回覆模板
2. 幫你設定 LINE 官方帳號的自動回覆（或教你設定）
3. 一週後檢查效果，調整 3 個回覆

## 不做清單
1. 不做訂單管理 — 你的 Excel 就夠用
2. 不做多平台 — 你 90% 客服在 LINE
3. 不做 AI 聊天 — 你要的是固定問答不是聊天機器人
4. 不做美化 / 品牌設計 — 回覆內容對就好

## 手工交付步驟
1. [30min] 跟客戶通話，列出他最常被問的 10 個問題
2. [60min] 寫好 10 組「問題+回覆」模板
3. [30min] 幫他在 LINE OA 設定自動回覆關鍵字
4. [30min] 測試 + 教他怎麼自己修改
5. [1週後 30min] 檢查使用情況，調整 3 個回覆
   合計：約 3 小時

## 定價
- 價格：NT$800（一次性）
- 邏輯：客服 SaaS 月費 2000，我的是一次性且只做核心
- 退費：7 天內不滿意全額退費

## Unit Economics
- 收入：NT$800
- 成本：3hr × NT$300/hr = NT$900
- 毛利：-NT$100（前期虧，學習成本）
- 自動化後：模板可重用，第 5 單起成本降到 1.5hr = NT$450 → 毛利 +NT$350

## 前 3 個客戶 + 訊息草稿
1. 小美 —— [LINE 訊息草稿]
2. 阿珍 —— [LINE 訊息草稿]
3. 社團認識的 Jenny —— [LINE 訊息草稿]
```
