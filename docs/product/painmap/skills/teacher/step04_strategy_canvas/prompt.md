# Step 04 策略圖設計 — 老師示範版 Prompt

> **版本**：老師示範版（課堂 live demo 專用）
> **對應技能**：`04_strategy_canvas.md` — Prompt 4A + Prompt 4B（紅隊）
> **示範場景**：小型電商賣家的重複客服勞動
>
> 老師說明：「我們在 Phase 3 拿到了破壞機會卡片：
> 低端破壞、月營收 3-10 萬的 Shopee 個人賣家、用只做 3 件事的極簡客服助手切入。
> 現在我們把這張卡片直接帶入 Strategy Canvas prompt，看看 AI 怎麼幫我們
> 畫出差異化曲線——然後立刻用紅隊 prompt 挑戰自己。」

---

## Prompt 4A：生成 Strategy Canvas（直接複製貼入 AI 聊天工具）

```text
你是一位策略圖（Strategy Canvas）設計師。
我要針對一個破壞式機會畫出策略圖。

## 我的資訊

**問題**：小型電商賣家每天花 2 小時重複回答相同客服問題（尺寸、運費、到貨時間）
**目標族群**：月營收 3-10 萬的 Shopee 個人賣家（一人公司，沒有客服人員）
**破壞模式**：低端破壞
**切入角度**：用「只做 3 件事的極簡客服助手」服務被客服 SaaS 過度服務的小賣家

**現有替代方案 / 競品**：
1. 某客服 SaaS（如 Omnichat、Crisp 等）：多平台整合、數據分析、AI 智能回覆，月費 NT$2,000+
2. 手動回 LINE：賣家自己每天打字回訊息，免費但耗時 1-2 小時/天
3. 不回覆（等太久直接流失）：零成本但直接損失訂單

---

請幫我：

### Step 1：定義功能維度

列出 6–8 個「客戶在選擇解法時真正在意的維度」。

規則：
- 用客戶語言，不是行業術語（「設定複雜度」→「多快能開始用」）
- 必須是可比較的（每個競品都能標高/中/低）
- 包含「價格」和「易用性」（幾乎所有場景都相關）
- 不要超過 8 個（太多 = 失焦）

### Step 2：標出競品位置

對每個替代方案，在每個維度標 高/中/低。
用表格呈現：

| 維度 | 定義 | 某客服SaaS | 手動回LINE | 不回覆 |
|------|------|-----------|-----------|--------|
| 價格 | 每月成本 | 高(NT$2000) | 低(時間成本) | 無 |
| ... | ... | ... | ... | ... |

### Step 3：建議「你的」差異化曲線

基於我的破壞模式和目標族群，建議我的曲線：
- 哪些維度應該拉到「高」（你的獨特優勢）
- 哪些維度應該壓到「低」（刻意不做）
- 每個建議都說明理由

**用 ASCII 圖呈現**：

```
維度：  月費  上手速  功能數  自動回  訂單追  多平台  中文化  可靠度
競品A：  ■■■   ■      ■■■■   ■■■    ■■■    ■■■    ■■     ■■■
手動：   ─    ■■■■    ─      ─      ■      ─     ■■■■    ■■
你的：   ■    ■■■■    ■      ■■■■   ─      ─     ■■■■    ■■■
```

### Step 4：差異化摘要

用一段話總結：
「我用 [什麼優勢] 服務 [什麼人]，刻意不做 [什麼]，
因為 [為什麼不做比做更有力]。」

### 規則
- 不打分數（高/中/低 是相對位置，不是分數）
- 你的曲線必須跟競品「形狀不同」，不是「全部更低」
- 「刻意不做」的維度必須有理由
- 如果競品資訊不足，標注「需要你實際使用 / 看評論才能確認」
```

## Prompt 4B：挑戰策略圖（紅隊模式）

> 老師說明：「拿到 4A 的輸出之後，不要馬上接受。
> 把完整輸出貼入下面這個 prompt，讓 AI 用五個問題打臉你自己的策略。
> 紅隊 prompt 的目的是在花錢做產品之前，先找出致命漏洞。」

```text
你是一位策略紅隊審查員。我剛畫了一張 Strategy Canvas，
請用以下 5 個問題挑戰我的策略。不要安慰我，直接說問題在哪。

我的 Strategy Canvas：

【貼入 Prompt 4A 的完整輸出】

---

問題情境補充：
- 目標族群：月營收 3-10 萬的 Shopee 個人賣家
- 破壞模式：低端破壞
- 切入角度：只做 3 件事的極簡客服助手（自動回常見問答、通知發貨、節假日暫停）

---

### 挑戰問題

1. **假低端陷阱**：我砍掉的功能（多平台整合、訂單追蹤、AI 智能客服），
   我的目標使用者真的不需要嗎？
   還是他們嘴上說不需要，實際用到時會抱怨？
   → 怎麼驗證？

2. **「更便宜」不等於「破壞」**：我的曲線是不是只是「全部壓低」？
   如果是，我不是破壞者，我只是便宜貨。
   → 哪些維度我應該比競品更高？

3. **護城河風險**：某客服 SaaS 如果也推出一個 NT$300 的低價版，我怎麼辦？
   我有什麼是他們做了低價版也複製不了的？

4. **目標客群是否真的被忽略**：大型客服 SaaS 真的不想做月營收 3-10 萬賣家的生意嗎？
   還是他們只是還沒做到這塊？（如果「還沒做到」→ 你沒有時間優勢）

5. **「不做」的代價**：在我「刻意不做」的功能中，有沒有哪一個
   在小賣家的使用者旅程中是「一旦需要就完全卡住」的關鍵功能？

### 規則
- 不安慰我（「你做得很好」= 無用輸出）
- 每個挑戰都給出「怎麼驗證」的具體方法
- 如果我的策略有致命缺陷，直說
```

## 老師備注

- 示範 4A 時強調：競品資訊的來源（官網定價頁、App Store 差評）；知道「大概做什麼」就夠了
- 讓學生先看 ASCII canvas 的「形狀」，而不是逐一讀數字——形狀不同才是關鍵
- 示範 4B 時強調：把 4A 的完整輸出貼入，讓 AI 有足夠上下文才能挑戰
- 收到兩個 prompt 的 AI 回覆後，請對照 `output_sample.md` 討論
- 告訴學生：如果紅隊讓你覺得整個策略有問題，這是好事——比做了 3 個月才發現便宜多了
