# Step 03 破壞式機會辨識 — 老師示範版 Prompt

> **版本**：老師示範版 — Phase 3（課堂 live demo 專用）
> **對應技能**：`03_disruption_scanner.md` — Prompt 3A
> **前置輸入**：Step 02 產出
> - 問題主題：「小型電商賣家的重複客服勞動」
> - 問題本質：「Shopee 個人賣家每天花 1-3 小時手動回覆重複問題（尺寸、運費、材質），導致無法把時間花在開發新品和行銷上」
> - 物理量畫布：✅具體的人（小美、阿珍、Jenny）／✅付錢的痛（花時間）／⚠️手工交付／✅收款
> - 現行替代方案：手動回 LINE、某客服 SaaS（NT$2000/月）、不回覆
>
> 老師說明：「我現在把 Step 02 的結論直接帶進來，特別是『現行替代方案』那欄，
> 這是 Phase 3 最重要的輸入。我們來看 AI 怎麼幫我掃描破壞式機會。」

---

## 示範用 Prompt（直接複製貼入 AI 聊天工具）

```text
你是一位破壞式創新分析師（基於 Clayton Christensen 框架）。
我會給你一個結構化的問題主題和現行替代方案。

請幫我分析這個問題有沒有破壞式切入的機會。

## 我的問題主題
Shopee 個人賣家每天花 1-3 小時手動回覆重複問題（尺寸、運費、材質），
導致無法把時間花在開發新品和行銷上。

## 目標族群
月營收 3-10 萬的 Shopee 個人賣家，商品品項不多（通常 10-30 款），
沒有專職客服人員，老闆本人兼任客服。
已知的具體人物：小美（手作飾品）、阿珍（家居小物）、Jenny（韓系服飾）。

## 現行替代方案
1. 手動回 LINE / Shopee 聊天室
   - 成本：每天約 1-3 小時
   - 最不滿的點：每天重複回答一模一樣的問題，像是機器人卻沒有機器人的效率

2. 某客服 SaaS（例如 Zendesk、Freshdesk 或本土類似產品）
   - 成本：約 NT$2000/月起跳
   - 最不滿的點：設定太複雜、功能太多用不到，感覺是設計給大公司的

3. 不回覆 / 直接忽略
   - 成本：潛在流失客人、差評風險
   - 最不滿的點：知道這樣不好，但實在沒時間處理

---

請輸出以下分析：

### 1. 替代方案分類

把每個現行替代方案歸類：

| 替代方案 | 類型 | 破壞訊號 |
|----------|------|----------|
| {{名稱}} | 昂貴工具 / 複雜手動 / 外包 / 免費但不足 / 忍受 / 無 | 低端 / 新市場 / 無 |

### 2. 低端破壞機會分析

如果有「昂貴工具 / 複雜手動 / 外包」類的替代方案：
- 這些使用者付了多少錢 / 時間？
- 他們真的用到所有功能嗎？哪些功能是「有但從沒用過」的？
- 如果砍掉那些不用的功能，只保留核心 3 個，他們會接受嗎？
- 砍完之後成本能降多少？

### 3. 新市場破壞機會分析

如果有「忍受 / 無 / 免費但不足」類的替代方案：
- 這些人為什麼沒有用現有工具？（太貴？不知道有？太複雜？）
- 他們是「不知道有這種東西」還是「知道但買不起」？
- 如果提供一個「不完美但終於有」的解法，他們會用嗎？
- 「終於有」的最低標準是什麼？（只需要做到什麼程度就比「沒有」好？）

### 4. 破壞機會摘要

用一張卡片格式總結：

破壞模式：低端破壞 / 新市場破壞 / 兩者皆有 / 無明顯機會
目標族群：[被過度服務的人 / 非消費者]
現行替代方案的最大弱點：[一句話]
你的切入角度：[一句話：用「更 ___」的方式服務「___ 的人」]

### 規則
- 不打分數、不預測成功率
- 如果看不出破壞機會，直說「目前看不出明顯的破壞切入點，原因是 ___」
  這不代表不能做，只代表需要換個角度或補更多資料
- 每個結論都標注「這是基於你提供的資料推測的，需要訪談驗證」
```

## 老師備注

- 示範時重點說明：**「現行替代方案」是這個 prompt 最重要的輸入**，寫得越具體，AI 的分析越有參考價值
- 提醒學生：AI 的分析是基於文字推測，不是市場調查結論——遇到「AI 說有機會但我不確定」，正確動作是「記下來，去訪談驗證」
- 收到 AI 回覆後，請對照 `output_sample.md` 討論：哪些 AI 說的你認同？哪些你有不同直覺？
- 這一步的核心教學目標不是讓學生「相信 AI 的結論」，而是學會**用 AI 的分析框架來組織自己的思考**
