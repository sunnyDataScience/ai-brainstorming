# PainMap 破壞式創新引導流程 — 使用指南

> **適用對象**：初學者、第一次嘗試從 0→1 打造產品的人
> **適用平台**：任何 AI 聊天工具（ChatGPT、Claude、Gemini、Copilot 等皆可）
> **所需時間**：完整流程約 3–5 小時（可分多天完成）
> **前置需求**：一個模糊的想法，或「我想做點什麼但不知道做什麼」的狀態

---

## 這套流程能幫你做什麼

把你腦中散亂的想法，一步步整理成：
1. 一個**結構化的真實問題**（不是空想）
2. 一張**策略圖**（知道要做什麼、更重要的是不做什麼）
3. 一份**可執行的 GTM 一頁紙**（明天就能開始行動）

**這套流程不會幫你**：打分數、預測成功率、寫商業計畫書、做投影片。
這些都是「代理動作」——看起來像進展，但不會讓你更接近第一塊錢。

---

## 流程全貌

```
Phase 1 — 痛點採集         「我看到了什麼？」
    ↓
Phase 2 — 問題本質拆解     「這些痛點背後的真正問題是什麼？」
    ↓
Phase 3 — 破壞式機會辨識   「哪裡有大公司不想做的角落？」
    ↓
Phase 4 — 策略圖設計       「我要做什麼？更重要的是不做什麼？」
    ↓
Phase 5 — 最小產品設計     「最小能賣的東西長什麼樣？」
    ↓
Phase 6 — 不對稱動機檢驗   「大公司會不會來搶？」
    ↓
Phase 7 — GTM 一頁紙       「明天第一步做什麼？」
```

---

## 怎麼使用

### 方法 A：按順序跑（推薦初學者）

1. 打開任意 AI 聊天工具
2. 從 `01_pain_collector.md` 開始
3. 把檔案中的 **Prompt** 複製貼上到聊天視窗
4. 按照指示填入你的內容（`{{雙大括號}}` 的地方換成你的資料）
5. 看 AI 的回覆，對照 **檢查點** 確認品質
6. 把產出存起來（Google Doc / Notion / 純文字都行）
7. 帶著產出進入下一個 Phase

### 方法 B：跳著跑（有經驗的人）

- 已經知道問題是什麼 → 跳過 Phase 1-2，從 Phase 3 開始
- 只想做 Strategy Canvas → 直接用 Phase 4
- 已經有產品雛形，想檢驗方向 → 從 Phase 6 開始

### 方法 C：反覆迭代

任何 Phase 的產出都不是「最終版」。你隨時可以：
- 帶著新的發現回到前面的 Phase 重跑
- 在 Phase 6 發現問題 → 回到 Phase 4 調整策略
- 跑完一次全流程後，隔一週帶著新資料再跑一次

## 每個 Phase 的檔案結構

每份 skill 檔案都有統一格式：

| 區段 | 說明 |
| :--- | :--- |
| **目標** | 這一步要達成什麼 |
| **你需要準備** | 開始前要有什麼（前一步的產出、你自己的素材等） |
| **Prompt** | 複製貼上到 AI 的完整 prompt（`{{}}` 處替換成你的內容） |
| **AI 回覆後你要做的事** | 拿到 AI 回覆後的人工步驟（AI 不能取代的部分） |
| **檢查點** | 確認產出品質的 checklist |
| **常見卡關與解法** | 做不下去時怎麼辦 |
| **產出範本** | 這一步完成後你應該有什麼 |
| **下一步** | 帶著產出去哪裡 |

## 核心紀律（貫穿整個流程）

### 1. 禁止分數

整個流程中，你和 AI 都不會產出任何「0–100 分」「成功率 X%」「可行性 A/B/C」。
分數是焦慮的來源，不是決策的依據。

### 2. 用「問題」不用「點子」

你不是在「驗證一個點子好不好」。你是在「看清一個問題的本質」。
語言會影響思考：「我的點子」讓你防衛；「這個問題」讓你客觀。

### 3. AI 做執行，你做判斷

AI 負責：整理資料、生成草稿、找出模式、提供範本。
你負責：決定做不做、選哪個方向、判斷這個人是不是真的在意。

**永遠不要問 AI「我該不該做這個？」**
改問「幫我整理這些痛點的結構」「幫我畫出競品的策略圖」。

### 4. 每一步都要有「具體的人」

不是「目標客群是 SME 老闆」，是「我認識的王大明，開早餐店的，每天花 2 小時手動記帳」。
如果你在任何 Phase 填不出具體的人 → 先去參與社群，認識真的人。

## 適用場景

| 你的狀態 | 建議路徑 |
| :--- | :--- |
| 「我完全不知道要做什麼」 | Phase 1 開始，用社群留言 / 日常觀察作為素材 |
| 「我有很多想法但不知道選哪個」 | Phase 1 全部倒進去 → Phase 2 結構化比較 |
| 「我知道問題是什麼，想找切入角度」 | Phase 3 開始 |
| 「我已經有產品，想檢查方向對不對」 | Phase 4 + Phase 6 |
| 「我什麼都做好了，缺一份 GTM 計畫」 | Phase 7 |

## 開始吧

打開 `01_pain_collector.md`，準備好你的素材（留言截圖、社群對話、腦中想法、筆記本），開始第一步。
