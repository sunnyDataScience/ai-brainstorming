# Phase 4 — 策略圖設計（Strategy Canvas）

> **目標**：畫出你和競品的功能曲線，明確「做什麼」和「不做什麼」
> **預估時間**：30–45 分鐘
> **前一步**：`03_disruption_scanner.md`（需要破壞機會卡片）
> **下一步**：`05_minimum_product.md`

---

## 你需要準備

- Phase 3 的破壞機會卡片（破壞模式 + 目標族群 + 替代方案 + 切入角度）
- 對現有替代方案 / 競品的基本了解（不需要深度研究，知道「他們大概做什麼」就夠）

---

## 背景知識（2 分鐘讀完）

Strategy Canvas 是藍海策略的核心工具，但我們用的是簡化版：

```
        高 │         ╱╲
功能      │    ╱╲  ╱  ╲        ← 競品 A（什麼都做，很貴）
程度      │   ╱  ╲╱    ╲
          │  ╱           ╲
        低 │╱              ╲
           └──┬──┬──┬──┬──┬──
             價格 易用 功能 速度 支援
                  性   數
```

**你要做的事**：
1. 列出 6–8 個客戶在意的「功能維度」
2. 標出競品在每個維度的位置
3. 畫出「你的曲線」——跟競品長得**不一樣**才對

**關鍵心法**：如果你的曲線和競品長得一樣但「全部更低」= 你只是更爛的版本。如果你的曲線和競品**形狀不同**（有些高、有些低）= 你在做不同的事。

## Prompt（複製貼上到任何 AI 聊天工具）

### Prompt 4A：生成 Strategy Canvas

```text
你是一位策略圖（Strategy Canvas）設計師。
我要針對一個破壞式機會畫出策略圖。

## 我的資訊

**問題**：{{Phase 2 的問題本質一句話}}
**目標族群**：{{破壞機會卡片的目標族群}}
**破壞模式**：{{低端破壞 / 新市場破壞}}
**切入角度**：{{破壞機會卡片的切入角度}}

**現有替代方案 / 競品**：
1. {{名稱}}：{{簡述做什麼、大約價位}}
2. {{名稱}}：{{簡述}}
3. {{名稱 / 或「手動處理」/ 或「不處理」}}

---

請幫我：

### Step 1：定義功能維度

列出 6–8 個「客戶在選擇解法時真正在意的維度」。

規則：
- 用客戶語言，不是行業術語（「設定複雜度」→「多快能開始用」）
- 必須是可比較的（每個競品都能標高/中/低）
- 包含「價格」和「易用性」（幾乎所有場景都相關）
- 不要超過 8 個（太多 = 失焦）

### Step 2：標出競品位置

對每個替代方案，在每個維度標 高/中/低。
用表格呈現：

| 維度 | 定義 | 競品A | 競品B | 手動處理 |
|------|------|-------|-------|----------|
| 價格 | 每月成本 | 高 | 中 | 低（時間成本高） |
| ... | ... | ... | ... | ... |

### Step 3：建議「你的」差異化曲線

基於我的破壞模式和目標族群，建議我的曲線：
- 哪些維度應該拉到「高」（你的獨特優勢）
- 哪些維度應該壓到「低」（刻意不做）
- 每個建議都說明理由

**用 ASCII 圖呈現**：

```
維度：  價格  易用  功能數  速度  支援  整合  客製  安全
競品A：  ■■■   ■    ■■■■  ■■    ■■■  ■■■  ■■■  ■■■
競品B：  ■■    ■■   ■■■   ■■■   ■■   ■■   ■■   ■■
你的：   ■    ■■■■   ■    ■■■■  ■■    ─    ─    ■■
```

### Step 4：差異化摘要

用一段話總結：
「我用 [什麼優勢] 服務 [什麼人]，刻意不做 [什麼]，
因為 [為什麼不做比做更有力]。」

### 規則
- 不打分數（高/中/低 是相對位置，不是分數）
- 你的曲線必須跟競品「形狀不同」，不是「全部更低」
- 「刻意不做」的維度必須有理由
- 如果競品資訊不足，標注「需要你實際使用 / 看評論才能確認」
```

### Prompt 4B：挑戰你的策略圖（紅隊模式）

產出策略圖後，用這個 prompt 自我挑戰：

```text
你是一位策略紅隊審查員。我剛畫了一張 Strategy Canvas，
請用以下 5 個問題挑戰我的策略：

我的 Strategy Canvas：
{{貼上 Prompt 4A 的完整輸出}}

### 挑戰問題

1. **假低端陷阱**：我砍掉的功能，我的目標使用者真的不需要嗎？
   還是他們嘴上說不需要，實際用到時會抱怨？
   → 怎麼驗證？

2. **「更便宜」不等於「破壞」**：我的曲線是不是只是「全部壓低」？
   如果是，我不是破壞者，我只是便宜貨。
   → 哪些維度我應該比競品更高？

3. **護城河風險**：競品如果也推出一個低價版，我怎麼辦？
   我有什麼是他們做了低價版也複製不了的？

4. **目標客群是否真的被忽略**：大公司真的不想做這群人的生意嗎？
   還是他們只是還沒做到？（如果「還沒做到」→ 你沒有時間優勢）

5. **「不做」的代價**：我「刻意不做」的功能中，有沒有哪一個
   在使用者旅程中是「一旦需要就完全卡住」的關鍵功能？

### 規則
- 不安慰我（「你做得很好」= 無用輸出）
- 每個挑戰都給出「怎麼驗證」的具體方法
- 如果我的策略有致命缺陷，直說
```

## AI 回覆後你要做的事

### 1. 調整策略圖

根據紅隊的挑戰，調整你的曲線。可能的調整：
- 把某個「刻意不做」改成「做基本版」
- 把某個「高」降為「中」（不需要做到最好）
- 發現一個被忽略的維度需要加上去

### 2. 確認「不做清單」

你的「不做清單」是整個策略最重要的部分。寫下來：

```markdown
## 我的 Strategic No（不做清單）

1. 不做 ___，因為：___
2. 不做 ___，因為：___
3. 不做 ___，因為：___

原則：不做清單必須比「要做清單」長。
如果你的「不做」比「要做」少 → 你砍得不夠。
```

## 常見卡關與解法

### 卡關 A：「我不夠了解競品，維度標不出來」

**解法**：花 20 分鐘做快速競品體驗。
- 去競品官網看功能頁和定價頁
- 看 Product Hunt / App Store / Google Play 的使用者評論（重點看差評）
- 問 AI：「幫我列出 {{競品名}} 的主要功能和使用者常見抱怨」

不需要深度研究。你只需要知道「他們大概做什麼、使用者大概抱怨什麼」。

### 卡關 B：「我的曲線跟競品長一樣，找不到差異」

**解法**：
1. 加一個競品沒有的維度（例如：「中文在地化」「一人公司友善度」「5 分鐘上手」）
2. 考慮你的目標族群跟競品的客戶是不是其實不一樣。不一樣的人在意不一樣的維度

### 卡關 C：「紅隊問題讓我覺得整個策略都有問題」

**這是好事。** 在你花錢做產品之前發現問題，比做了 3 個月才發現便宜太多了。
回到 Phase 3 重新選擇切入角度，或回到 Phase 2 換一個問題主題。
**每次重來都讓你更接近正確答案。**

## 產出範本

```markdown
# Strategy Canvas — {{日期}}

## 問題：小型電商賣家的重複客服勞動
## 目標：月營收 3-10 萬的 Shopee 個人賣家

## 功能維度

| 維度 | 定義 | 某客服SaaS | LINE手動 | 我的產品 |
|------|------|-----------|---------|---------|
| 價格 | 月費 | 高(2000) | 低(0) | 低(300) |
| 上手速度 | 幾分鐘開始用 | 低(要設定) | 高(馬上) | 高(5分鐘) |
| 功能數 | 能做多少事 | 高(50+) | 低(只能打字) | 低(只做3件) |
| 常見問答 | 自動回覆常見問題 | 高 | 無 | 高 |
| 訂單追蹤 | 管理出貨 | 高 | 低(Excel) | 無 |
| 多平台 | 支援蝦皮+LINE+FB | 高 | 低(只LINE) | 低(只LINE) |
| 中文在地化 | 台灣用語 | 中 | 高(自己寫) | 高 |

## 差異化摘要
我用「5 分鐘上手、只做常見問答自動回覆」服務被客服 SaaS
過度服務的小賣家。刻意不做訂單追蹤、不做多平台、不做 50 個功能。
因為小賣家只需要「不用再重複回答尺寸和運費」這一件事。

## Strategic No（不做清單）
1. 不做訂單追蹤 — 小賣家用 Excel 就夠了，這不是他們的痛
2. 不做多平台整合 — 小賣家 90% 客服在 LINE，做蝦皮整合是工程浪費
3. 不做 AI 智能客服 — 小賣家要的是「固定問答」不是「AI 聊天」
4. 不做數據分析報表 — 小賣家看不懂也不需要
```
