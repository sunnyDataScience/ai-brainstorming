## 背景知識（2 分鐘讀完）

破壞式創新的核心觀念很簡單：

**你不需要比大公司更強。你需要找到大公司「不想做」的地方。**

大公司不想做的原因通常是：
- 這個客群太小，不值得他們的銷售團隊花時間
- 這個價位太低，會拉低他們的平均利潤
- 要砍掉功能才能服務這群人，但企業客戶不允許砍

兩種破壞模式：

| 模式 | 一句話 | 你找的是什麼人 |
| :--- | :--- | :--- |
| **低端破壞** | 現有工具太貴 / 太複雜，你提供「夠用的簡單版」 | 被過度服務的人（付太多錢、用不到那些功能） |
| **新市場破壞** | 根本沒有工具，你提供「終於有了」 | 非消費者（想解決但買不起 / 不知道怎麼解決） |

```text
### 1. 替代方案分類

把每個現行替代方案歸類：

| 替代方案 | 類型 | 破壞訊號 |
|----------|------|----------|
| {{名稱}} | 昂貴工具 / 複雜手動 / 外包 / 免費但不足 / 忍受 / 無 | 低端 / 新市場 / 無 |

### 2. 低端破壞機會分析

如果有「昂貴工具 / 複雜手動 / 外包」類的替代方案：
- 這些使用者付了多少錢 / 時間？
- 他們真的用到所有功能嗎？哪些功能是「有但從沒用過」的？
- 如果砍掉那些不用的功能，只保留核心 3 個，他們會接受嗎？
- 砍完之後成本能降多少？

### 3. 新市場破壞機會分析

如果有「忍受 / 無 / 免費但不足」類的替代方案：
- 這些人為什麼沒有用現有工具？（太貴？不知道有？太複雜？）
- 他們是「不知道有這種東西」還是「知道但買不起」？
- 如果提供一個「不完美但終於有」的解法，他們會用嗎？
- 「終於有」的最低標準是什麼？（只需要做到什麼程度就比「沒有」好？）

### 4. 破壞機會摘要

用一張卡片格式總結：

```
破壞模式：低端破壞 / 新市場破壞 / 兩者皆有 / 無明顯機會
目標族群：[被過度服務的人 / 非消費者]
現行替代方案的最大弱點：[一句話]
你的切入角度：[一句話：用「更 ___」的方式服務「___ 的人」]
```

### 規則
- 不打分數、不預測成功率
- 如果看不出破壞機會，直說「目前看不出明顯的破壞切入點，原因是 ___」
  這不代表不能做，只代表需要換個角度或補更多資料
- 每個結論都標注「這是基於你提供的資料推測的，需要訪談驗證」
```

## AI 回覆後你要做的事

### 1. 對照你的真實認知

AI 的分析是基於你提供的文字。但你可能知道一些沒寫出來的事。

問自己：
- **AI 說的「overserved 族群」，我認識這樣的人嗎？** → 如果認識，在物理量畫布補上
- **AI 說的「非消費者」，我見過嗎？** → 如果見過，記下來
- **AI 的分析有沒有跟我的直覺矛盾的地方？** → 矛盾不代表 AI 錯，但值得記下來去訪談驗證

### 2. 選定破壞模式

你可能同時有低端和新市場的機會。但 **一次只做一個**。

選擇原則：
- **你認識的人集中在哪一群？** → 選那群（因為驗證最快）
- **哪一群的現行解法最爛？** → 選那群（因為切入最容易）
- **不確定？** → 選低端破壞（因為有明確的競品可以對比）

## 常見卡關與解法

### 卡關 A：「AI 說看不出明顯的破壞機會」

**不要慌。** 可能的原因：
1. 你的替代方案資訊不夠 → 回去補「他們現在花多少錢 / 時間」的具體資料
2. 這個領域真的沒有被過度服務的人 → 考慮轉向「新市場破壞」：有沒有想要但買不起的人？
3. 這個問題可能不適合「破壞式」但適合「差異化」→ 直接跳到 Phase 4 的 Strategy Canvas，換個角度看

### 卡關 B：「低端和新市場都有機會，選不了」

**選你認識最多人的那個。** 因為接下來你要做 Strategy Canvas 和 GTM，認識的人越多 = 驗證越快 = 犯錯成本越低。

### 卡關 C：「我的競品是免費工具（像 Google Sheet），怎麼低端破壞？」

**免費工具的真實成本是時間。** 你的切入不是「比免費更便宜」，而是「比免費省時間」。問你的目標使用者：「你花在 Google Sheet 上的 2 小時值多少錢？」如果答案 > 0，你就有收費空間。

```markdown
## 破壞機會卡片

- **破壞模式**：低端破壞
```
