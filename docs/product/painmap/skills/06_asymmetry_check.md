# Phase 6 — 不對稱動機檢驗

> **目標**：確認大公司不會（或不想）搶你的生意——這是發送預售訊息前的最後檢查
> **預估時間**：15–20 分鐘
> **前一步**：`05_minimum_product.md`（需要最小產品設計 + 定價）
> **下一步**：`07_gtm_one_pager.md`

---

## 你需要準備

- Phase 5 的最小產品設計（核心價值主張 + 定價 + 目標族群）
- Phase 4 的 Strategy Canvas（你和競品的曲線）
- Phase 3 的破壞機會卡片（破壞模式 + 替代方案）

---

## 為什麼需要這一步（2 分鐘讀完）

很多小產品死掉，不是因為做得不好，是因為大公司輕鬆就能複製。

Christensen 的核心洞察是：**破壞成功不是因為你多強，而是因為大公司「不想」回應你。**

大公司不想回應的原因：
- 你的市場太小，他們的銷售團隊不屑
- 你的價格太低，他們跟進會拉低自己的利潤
- 要服務你的客群，他們必須砍掉功能，但企業客戶不允許

**這一步就是檢查：這些條件成立嗎？**

### Q2：自我蠶食

競品推出低價版會傷到自己嗎？

- 競品現在的定價結構是什麼？
- 如果競品推出 NT${{我的價格}} 的版本，他們的高端客戶會不會降級？
- 這個降級對競品的營收影響有多大？

結論：競品「會 / 不會」因為害怕自我蠶食而不推低價版。
如何驗證：___

### Q3：功能裁減

競品能不能砍功能來跟我競爭？

- 競品的企業客戶依賴哪些「我不做」的功能？
- 如果競品砍掉那些功能做一個簡單版，企業客戶會不會抗議？
- 競品的工程團隊願不願意維護兩個版本（完整版 + 簡單版）？

結論：競品「能 / 不能」輕易砍功能來跟我競爭。
如何驗證：___

### Q4：銷售注意力

競品的銷售團隊會重視我的客群嗎？

- 競品的業務一單做多大？（一單 NT$2 萬 vs 我的一單 NT$800）
- 業務會不會覺得「做這種小單不值得」？
- 競品有沒有 self-serve（自助註冊）的管道可以低成本獲取我的客群？

結論：競品的銷售團隊「會 / 不會」花力氣搶我的小客戶。

### 總結

| 問題 | 結論 | 風險等級 |
|------|------|---------|
| Q1 利潤保護 | 會/不會行動 | 安全 ✅ / 注意 ⚠️ |
| Q2 自我蠶食 | 會/不會推低價 | 安全 ✅ / 注意 ⚠️ |
| Q3 功能裁減 | 能/不能砍功能 | 安全 ✅ / 注意 ⚠️ |
| Q4 銷售注意力 | 會/不會搶 | 安全 ✅ / 注意 ⚠️ |

**整體評估**：
- 4 個 ✅ → 強不對稱動機，放心進入市場
- 3 個 ✅ + 1 個 ⚠️ → 可行，但需要速度——盡快拿到第一塊錢
- 2 個 ✅ + 2 個 ⚠️ → 有風險，考慮調整目標客群或定價
- ≤ 1 個 ✅ → 不是破壞式機會，回 Phase 3 重新選擇

### 規則
- 不打分數，不預測成功率
- 「安全 / 注意」不是分數，是事實判斷
- 每個結論都附「如何驗證」——因為這些都是假設
- 如果資料不足，直說「需要了解更多競品的營收結構 / 客戶構成」

| 結果 | 行動 |
| :--- | :--- |
| 3-4 個 ✅ | 進入 Phase 7，準備 GTM 一頁紙，發預售訊息 |
| 2 個 ✅ | 回顧你的目標客群和定價，考慮調整到「更小、更被忽略」的族群 |

### 3. 如果需要調整

常見調整方向：
- **目標客群太大** → 縮小到更被忽略的子群（不是「電商賣家」，是「月營收 < 5 萬的手作媽媽」）
- **定價太高** → 降到競品不屑跟進的程度
- **功能太接近競品** → 砍更多功能，讓你的產品跟競品「長得更不一樣」

## 常見卡關與解法

### 卡關 A：「我不知道競品的營收結構」

**你不需要精確數字。** 用以下方式估計：
- 看定價頁 → 企業版多少錢、個人版多少錢
- 看 LinkedIn → 他們的銷售團隊有多大？大 = 靠高單價生意
- 看 Crunchbase / 新聞 → 融資多少？年營收估計？
- 問 AI：「根據 {{競品名}} 的定價和團隊規模，估計他們的主要營收來源是什麼？」

### 卡關 B：「4 個都是 ⚠️」

**不要氣餒。** 這代表你選的不是「被忽略的角落」，而是「大家都在搶的市場」。
兩個選擇：
1. **縮小** — 找到更小、更被忽略的子群
2. **轉向** — 回到 Phase 2，選另一個問題主題

**這不是失敗，這是省下了做錯方向的 3 個月。**

### 卡關 C：「我的競品是免費工具 / 開源工具」

免費工具的「銷售團隊」是社群。問自己：
- 這個社群會不會針對你的客群做特別的推廣？（通常不會）
- 你的客群有能力自己設定 / 使用這個免費工具嗎？（如果不能，你的「手工幫他設定好」就是價值）
- 免費工具的維護者有商業動機做你在做的事嗎？（如果沒有 → ✅）
